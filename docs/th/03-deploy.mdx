ไฟล์ Terraform ทั้งหมดอยู่ในไดเรกทอรี [`terraform/`](https://github.com/f5xc-salesdemos/cdn-simulator/tree/main/terraform) โคลนรีโพซิทอรีและปรับใช้งานได้โดยตรง:

```bash
git clone https://github.com/f5xc-salesdemos/cdn-simulator.git
cd cdn-simulator/terraform
cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars with your Azure subscription ID and origin server
```

## การตั้งค่า Terraform

### โครงสร้างไฟล์ Terraform

ไดเรกทอรี terraform ประกอบด้วย 9 ไฟล์ตามมาตรฐาน Demo Resource Standard:

- `versions.tf` — ข้อจำกัดเวอร์ชันของ Terraform และ provider (azurerm ~> 4.0, azuread ~> 3.0)
- `providers.tf` — การตั้งค่า Azure RM และ Azure AD provider
- `data.tf` — Azure AD data sources สำหรับการระบุตัวตนผู้ปรับใช้งานโดยอัตโนมัติ
- `locals.tf` — การระบุตัวตนผู้ปรับใช้งาน, การตั้งชื่อทรัพยากรตาม Azure Cloud Adoption Framework, แท็กมาตรฐาน
- `main.tf` — Resource group (ชื่อ `rg-cdn-simulator-{environment}-{deployer}`)
- `variables.tf` — ตัวแปร input ทั้งหมด (จำเป็น 3 ตัว, ไม่บังคับ 8 ตัว)
- `network.tf` — VNet (10.100.0.0/16), subnet, public IP, NSG (พอร์ต 22/80/443), NIC
- `vm.tf` — Ubuntu 24.04 VM พร้อม cloud-init ผ่าน templatefile()
- `outputs.tf` — outputs 17 รายการ (มาตรฐาน 15 + เฉพาะคอมโพเนนต์ 2)

`variables.tf` กำหนดตัวแปร input 11 ตัวจัดเป็นหมวด General, Compute และ Component-Specific ตัวระบุ `deployer` จะถูกระบุโดยอัตโนมัติจากบัญชี Azure AD ของคุณ — คุณเพียงแค่ต้องตั้งค่า `subscription_id`, `origin_server` และ `origin_host`:

```hcl file=./_data/variables.tf
```

### โครงสร้างพื้นฐานเครือข่าย

`network.tf` สร้าง VNet, subnet, public IP, NSG (พอร์ต 22/80/443) และ NIC:

```hcl file=./_data/network.tf
```

### เครื่องเสมือนพร้อม Cloud-Init

`vm.tf` สร้าง Ubuntu 24.04 VM เส้นทาง SSH public key จะถูกขยายผ่าน `pathexpand()` เพื่อจัดการกับ `~` เทมเพลต Cloud-init จะรับตัวแปร `origin_server` และ `origin_host`:

```hcl file=./_data/vm.tf
```

### การจัดเตรียมด้วย Cloud-Init

`cloud-init.yaml` จัดเตรียม VM พร้อมการปรับแต่ง kernel, systemd limits, NGINX พร้อมการตั้งค่าที่ปรับแต่งประสิทธิภาพ, cache keys zone ขนาด 128 MB, upstream keepalive pool, gzip compression และ CDN vendor headers กว่า 67 รายการ ไลบรารี helper แบบใช้ร่วมกัน (`/usr/local/lib/cloud-init-helpers.sh`) มี retry logic และ progress logging ไปยัง `/var/log/cloud-init-progress.log`

Cloud-init ใช้ตัวแปรเทมเพลต Terraform: `${origin_server}` และ `${origin_host}` สำหรับการตั้งค่า upstream ตัวแปร NGINX เช่น `${request_id}` จะถูก escape เป็น `$${request_id}` ใน Terraform templatefile

```yaml file=./_data/cloud-init.yaml
```

### Outputs

`outputs.tf` แสดง outputs 17 รายการตามมาตรฐาน Demo Resource Standard — outputs มาตรฐาน 15 รายการที่ใช้ร่วมกันโดย demo resources ทั้งหมด (`deployer`, `public_ip`, `private_ip`, `ssh_command`, `resource_group_name`, `vm_name`, `nsg_name`, `vnet_name`, `subnet_id`, `component`, `environment`, `resource_group_id`, `vm_id`, `nsg_id`, `location`) บวกกับ outputs เฉพาะคอมโพเนนต์อีก 2 รายการ (`edge_url`, `health_check_url`):

```hcl file=./_data/outputs.tf
```

### ตัวอย่างไฟล์ตัวแปร

คัดลอก `terraform.tfvars.example` เป็น `terraform.tfvars` และกรอกค่าของคุณ `.gitignore` จะยกเว้น `terraform.tfvars` เพื่อป้องกันการ commit ข้อมูลรับรอง:

```hcl file=./_data/terraform.tfvars.example
```

## การปรับใช้งาน

```bash
# Initialize Terraform
terraform init

# Review the plan
terraform plan

# Apply
terraform apply
```

Terraform จะแสดง public IP, คำสั่ง SSH และ edge URL หลังจากปรับใช้งานสำเร็จ

## ขนาด VM

VM ซีรีส์ F-series ที่ปรับแต่งสำหรับการคำนวณได้รับการแนะนำสำหรับ workload NGINX proxy ที่ใช้ CPU เป็นหลัก ค่าเริ่มต้น `Standard_F4s_v2` (4 vCPU, 8 GiB) เหมาะสำหรับการใช้งาน lab และ demo สามารถแทนที่ตัวแปร `vm_size` สำหรับสถานการณ์การทดสอบโหลดหรือการวัดประสิทธิภาพในการผลิต การปรับแต่ง kernel ของ cloud-init และการตั้งค่า NGINX จะปรับขนาดโดยอัตโนมัติตามจำนวน vCPU (`worker_processes auto`)

## หลังการปรับใช้งาน

หลังจาก `terraform apply` เสร็จสิ้น ให้รอ 2-3 นาทีเพื่อให้ cloud-init ติดตั้งและตั้งค่า NGINX เสร็จสมบูรณ์ ตรวจสอบ health endpoint:

```bash
curl -s "http://$(terraform output -raw public_ip)/health" | jq .
```

ผลลัพธ์ที่คาดหวัง:

```json
{
  "status": "healthy",
  "component": "cdn-edge",
  "engine": "nginx",
  "vendor_profiles": [
    "akamai",
    "cloudflare",
    "cloudfront",
    "fastly",
    "azure-front-door"
  ]
}
```

## การเชื่อมต่อจากคอมโพเนนต์ต้นทาง

CDN simulator ต้องการ origin server ที่ปรับใช้งานแล้ว ใช้ terraform outputs ของ origin server เพื่อกรอกตัวแปรที่จำเป็น:

```bash
cd ../origin-server/terraform
origin_ip=$(terraform output -raw public_ip)

cd ../../cdn-simulator/terraform
cat > terraform.tfvars <<EOF
subscription_id = "your-subscription-id"
origin_server   = "http://${origin_ip}"
origin_host     = "${origin_ip}:80"
EOF
```

| ตัวแปรที่จำเป็น | แหล่งที่มา | รูปแบบ |
|---|---|---|
| `origin_server` | Output `public_ip` ของ Origin server | `http://<ip>` (รวม scheme) |
| `origin_host` | Output `public_ip` ของ Origin server | `<ip>:80` (ไม่มี scheme, รวมพอร์ต) |

ดำเนินการต่อที่ [การตั้งค่า NGINX](../04-nginx-config/) สำหรับตัวเลือกการปรับแต่ง หรือ [การตรวจสอบ](../05-verify/) สำหรับการทดสอบแคช
