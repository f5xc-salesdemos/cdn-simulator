ไฟล์ Terraform ทั้งหมดอยู่ในไดเรกทอรี [`terraform/`](https://github.com/f5xc-salesdemos/cdn-simulator/tree/main/terraform) โคลน repository และปรับใช้งานได้โดยตรง:

```bash
git clone https://github.com/f5xc-salesdemos/cdn-simulator.git
cd cdn-simulator/terraform
cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars with your Azure subscription ID and origin server
```

## การกำหนดค่า Terraform

### โครงสร้างไฟล์ Terraform

ไดเรกทอรี terraform ประกอบด้วยไฟล์ 9 ไฟล์ตามมาตรฐาน ทรัพยากรสาธิต:

- `versions.tf` — ข้อจำกัดเวอร์ชัน Terraform และ provider (azurerm ~> 4.0, azuread ~> 3.0)
- `providers.tf` — การกำหนดค่า Azure RM และ Azure AD provider
- `data.tf` — แหล่งข้อมูล Azure AD สำหรับการแก้ไข deployer อัตโนมัติ
- `locals.tf` — การแก้ไข Deployer, การตั้งชื่อทรัพยากรตาม Azure Cloud Adoption Framework, แท็กมาตรฐาน
- `main.tf` — Resource group (ชื่อ `rg-cdn-simulator-{environment}-{deployer}`)
- `variables.tf` — ตัวแปร input ทั้งหมด (3 ตัวที่จำเป็น, 8 ตัวที่ไม่บังคับ)
- `network.tf` — VNet (10.100.0.0/16), subnet, public IP, NSG (พอร์ต 22/80/443), NIC
- `vm.tf` — Ubuntu 24.04 VM พร้อม cloud-init ผ่าน templatefile()
- `outputs.tf` — 17 outputs (15 มาตรฐาน + 2 เฉพาะส่วนประกอบ)

`variables.tf` กำหนดตัวแปร input จำนวน 11 ตัว จัดแบ่งเป็นส่วน General, Compute, และ Component-Specific ตัวระบุ `deployer` จะถูกแก้ไขอัตโนมัติจากบัญชี Azure AD ของคุณ — คุณจำเป็นต้องตั้งค่าเพียง `subscription_id`, `origin_server`, และ `origin_host`:

```hcl file=./_data/variables.tf
```

### โครงสร้างพื้นฐานเครือข่าย

`network.tf` สร้าง VNet, subnet, public IP, NSG (พอร์ต 22/80/443), และ NIC:

```hcl file=./_data/network.tf
```

### Virtual Machine พร้อม Cloud-Init

`vm.tf` สร้าง Ubuntu 24.04 VM โดย path ของ SSH public key จะถูกขยายผ่าน `pathexpand()` เพื่อรองรับ `~` เทมเพลต Cloud-init จะรับตัวแปร `origin_server` และ `origin_host`:

```hcl file=./_data/vm.tf
```

### การจัดเตรียมด้วย Cloud-Init

`cloud-init.yaml` จัดเตรียม VM ด้วยการปรับแต่ง kernel, systemd limits, NGINX พร้อมการกำหนดค่าที่ปรับแต่งประสิทธิภาพ, cache keys zone ขนาด 128 MB, upstream keepalive pool, gzip compression, และ CDN vendor headers กว่า 67 รายการ ไลบรารีตัวช่วยที่ใช้ร่วมกัน (`/usr/local/lib/cloud-init-helpers.sh`) มีตรรกะการลองใหม่และการบันทึกความคืบหน้าไปยัง `/var/log/cloud-init-progress.log`

Cloud-init ใช้ตัวแปรเทมเพลต Terraform ได้แก่ `${origin_server}` และ `${origin_host}` สำหรับการกำหนดค่า upstream ตัวแปร NGINX เช่น `${request_id}` จะถูก escape เป็น `$${request_id}` ใน Terraform templatefile

```yaml file=./_data/cloud-init.yaml
```

### Outputs

`outputs.tf` เปิดเผย 17 outputs ตามมาตรฐาน ทรัพยากรสาธิต — 15 outputs มาตรฐานที่ใช้ร่วมกันโดยทรัพยากรสาธิตทั้งหมด (`deployer`, `public_ip`, `private_ip`, `ssh_command`, `resource_group_name`, `vm_name`, `nsg_name`, `vnet_name`, `subnet_id`, `component`, `environment`, `resource_group_id`, `vm_id`, `nsg_id`, `location`) บวกกับ 2 outputs เฉพาะส่วนประกอบ (`edge_url`, `health_check_url`):

```hcl file=./_data/outputs.tf
```

### ตัวอย่างไฟล์ Variables

คัดลอก `terraform.tfvars.example` ไปยัง `terraform.tfvars` และกรอกค่าของคุณ `.gitignore` จะยกเว้น `terraform.tfvars` เพื่อป้องกันการ commit ข้อมูลรับรอง:

```hcl file=./_data/terraform.tfvars.example
```

## การปรับใช้งาน

```bash
# Initialize Terraform
terraform init

# Review the plan
terraform plan

# Apply
terraform apply
```

Terraform จะแสดง public IP, คำสั่ง SSH, และ edge URL หลังจากปรับใช้งานสำเร็จ

## ขนาด VM

F-series compute-optimized VM แนะนำสำหรับ workload NGINX proxy ที่ใช้ CPU เป็นหลักนี้ ค่าเริ่มต้น `Standard_F4s_v2` (4 vCPU, 8 GiB) เหมาะสำหรับการใช้งานในห้องปฏิบัติการและการสาธิต สามารถแทนที่ตัวแปร `vm_size` สำหรับสถานการณ์การทดสอบโหลดหรือการวัดประสิทธิภาพในสภาพแวดล้อมการผลิต การปรับแต่ง kernel ผ่าน cloud-init และการกำหนดค่า NGINX จะปรับขนาดโดยอัตโนมัติตามจำนวน vCPU (`worker_processes auto`)

## หลังการปรับใช้งาน

หลังจาก `terraform apply` เสร็จสมบูรณ์ ให้รอ 2-3 นาทีเพื่อให้ cloud-init ติดตั้งและกำหนดค่า NGINX เสร็จสิ้น ตรวจสอบ health endpoint:

```bash
curl -s "http://$(terraform output -raw public_ip)/health" | jq .
```

การตอบสนองที่คาดหวัง:

```json
{
  "status": "healthy",
  "component": "cdn-edge",
  "engine": "nginx",
  "vendor_profiles": [
    "akamai",
    "cloudflare",
    "cloudfront",
    "fastly",
    "azure-front-door"
  ]
}
```

## การเชื่อมต่อจากส่วนประกอบ Upstream

ตัวจำลอง CDN ต้องการ origin server ที่ปรับใช้งานแล้ว ใช้ terraform outputs ของ origin server เพื่อกำหนดค่าตัวแปรที่จำเป็น:

```bash
cd ../origin-server/terraform
origin_ip=$(terraform output -raw public_ip)

cd ../../cdn-simulator/terraform
cat > terraform.tfvars <<EOF
subscription_id = "your-subscription-id"
origin_server   = "http://${origin_ip}"
origin_host     = "${origin_ip}:80"
EOF
```

| ตัวแปรที่จำเป็น | แหล่งที่มา | รูปแบบ |
|---|---|---|
| `origin_server` | output `public_ip` ของ Origin server | `http://<ip>` (ระบุ scheme) |
| `origin_host` | output `public_ip` ของ Origin server | `<ip>:80` (ไม่มี scheme, ระบุพอร์ต) |

ดำเนินการต่อไปยัง [การกำหนดค่า NGINX](../04-nginx-config/) สำหรับตัวเลือกการปรับแต่ง หรือ [การตรวจสอบ](../05-verify/) สำหรับการทดสอบ cache
