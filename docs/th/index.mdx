## สิ่งที่ได้รับ

- **[Edge Node เดียว](01-overview/)** — VM Ubuntu 24.04 หนึ่งเครื่องที่รัน NGINX เป็น caching reverse proxy ซึ่งเป็น CDN edge ขั้นต่ำที่ใช้งานได้
- **[การปรับใช้ด้วย Terraform](03-deploy/)** — โครงสร้างพื้นฐาน Azure แบบ infrastructure as code ครบถ้วน: VNet, PIP, NSG และ VM พร้อมการจัดเตรียมด้วย cloud-init
- **[เฮดเดอร์ CDN Vendor 67+ รายการ](04-nginx-config/)** — จำลองเฮดเดอร์คำขอจาก Akamai, Cloudflare, CloudFront, Fastly และ Azure Front Door รวมถึง True-Client-IP, CF-Connecting-IP, geolocation, device detection, JA3/JA4 fingerprints และ bot scores
- **[การรวมระบบกับ Origin](06-integrate/)** — กำหนดค่า edge ให้ส่งต่อ cache miss ไปยัง F5 XC HTTP load balancer ในฐานะ origin
- **[การตรวจสอบ Cache](05-verify/)** — ตรวจสอบพฤติกรรม HIT/MISS ด้วยเฮดเดอร์ X-Cache-Status และการตรวจสอบ NGINX cache

## อ้างอิงด่วน

หลังจากปรับใช้แล้ว CDN edge node จะเปิดให้เข้าถึงผ่าน:

| Endpoint | Method | วัตถุประสงค์ |
|---|---|---|
| `http://<PUBLIC_IP>/health` | GET | Health check — ส่งคืน JSON พร้อมสถานะส่วนประกอบและ vendor profiles |
| `http://<PUBLIC_IP>/` | GET | CDN proxy — แคชและส่งต่อคำขอไปยัง origin server |
| `http://<PUBLIC_IP>/<any-path>` | GET | CDN proxy — เส้นทางทั้งหมดจะถูก proxy พร้อมการแคชและเฮดเดอร์ vendor |

เฮดเดอร์ตอบกลับที่เพิ่มในทุกการตอบกลับที่ผ่าน proxy:

| เฮดเดอร์ | ตัวอย่าง | วัตถุประสงค์ |
|---|---|---|
| `X-Cache-Status` | `HIT` หรือ `MISS` | การตอบกลับถูกให้บริการจาก cache หรือไม่ |
| `X-CDN-Edge` | `cdn-simulator` | ระบุ edge node นี้ |
| `X-CDN-POP` | `SJC` | รหัส Point of Presence ที่จำลอง |
| `X-Served-By` | `cache-sjc3120-SJC` | ตัวระบุ cache node ที่จำลอง |
| `X-Request-ID` | UUID | ตัวระบุเฉพาะต่อคำขอ |
