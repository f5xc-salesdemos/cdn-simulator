## สิ่งที่ระบบนี้มอบให้

- **[Edge Node เดี่ยว](01-overview/)** — VM Ubuntu 24.04 หนึ่งเครื่องที่รัน NGINX เป็น caching reverse proxy ซึ่งเป็น CDN edge ขั้นต่ำที่ใช้งานได้
- **[การ Deploy ด้วย Terraform](03-deploy/)** — โครงสร้างพื้นฐาน Azure แบบ infrastructure as code ครบถ้วน: VNet, PIP, NSG และ VM พร้อมการ provisioning ด้วย cloud-init
- **[CDN Vendor Headers มากกว่า 67 รายการ](04-nginx-config/)** — จำลอง request headers ของ Akamai, Cloudflare, CloudFront, Fastly และ Azure Front Door รวมถึง True-Client-IP, CF-Connecting-IP, geolocation, device detection, JA3/JA4 fingerprints และ bot scores
- **[การเชื่อมต่อกับ Origin](06-integrate/)** — กำหนดค่า edge เพื่อส่งต่อ cache miss ไปยัง F5 XC HTTP load balancer ในฐานะ origin
- **[การตรวจสอบ Cache](05-verify/)** — ตรวจสอบพฤติกรรม HIT/MISS ด้วย X-Cache-Status headers และการตรวจสอบ NGINX cache

## ข้อมูลอ้างอิงอย่างรวดเร็ว

หลังจาก deploy แล้ว CDN edge node จะเปิดให้เข้าถึงได้ดังนี้:

| Endpoint | Method | วัตถุประสงค์ |
|---|---|---|
| `http://<PUBLIC_IP>/health` | GET | Health check — ส่งคืน JSON พร้อมสถานะของคอมโพเนนต์และ vendor profiles |
| `http://<PUBLIC_IP>/` | GET | CDN proxy — แคชและส่งต่อ request ไปยัง origin server |
| `http://<PUBLIC_IP>/<any-path>` | GET | CDN proxy — ทุก path จะถูก proxy พร้อม caching และ vendor headers |

Response headers ที่เพิ่มเข้าไปในทุก proxied response:

| Header | ตัวอย่าง | วัตถุประสงค์ |
|---|---|---|
| `X-Cache-Status` | `HIT` หรือ `MISS` | บ่งบอกว่า response ถูกส่งจาก cache หรือไม่ |
| `X-CDN-Edge` | `cdn-simulator` | ระบุ edge node นี้ |
| `X-CDN-POP` | `SJC` | รหัส Point of Presence จำลอง |
| `X-Served-By` | `cache-sjc3120-SJC` | ตัวระบุ cache node จำลอง |
| `X-Request-ID` | UUID | ตัวระบุเฉพาะสำหรับแต่ละ request |
