หน้านี้ครอบคลุมสองขั้นตอนของการเชื่อมต่อ:

1. **การเชื่อมต่อโดยตรง** — CDN edge ส่งต่อไปยัง origin server โดยตรง (การทดสอบพื้นฐาน)
2. **การแทรก F5 XC** — F5 XC HTTP load balancer ถูกแทรกระหว่าง CDN และ origin (การสาธิตด้านความปลอดภัย)

เริ่มต้นด้วยการเชื่อมต่อโดยตรงเพื่อสร้างพื้นฐาน จากนั้นแทรก F5 XC เมื่อพร้อม

## ข้อมูลอ้างอิง Origin Server

คอมโพเนนต์แล็บ [origin-server](https://f5xc-salesdemos.github.io/origin-server/) ให้บริการเว็บแอปพลิเคชันที่มีช่องโหว่สำหรับการทดสอบความปลอดภัย

| คุณสมบัติ | ค่า |
|---|---|
| เอกสาร | [f5xc-salesdemos.github.io/origin-server](https://f5xc-salesdemos.github.io/origin-server/) |
| Repository | [github.com/f5xc-salesdemos/origin-server](https://github.com/f5xc-salesdemos/origin-server) |
| พอร์ตเริ่มต้น | 80 |
| Health Check | `GET /health` |

### แอปพลิเคชันที่มีให้บริการ

ดึงรายการแอปพลิเคชันปัจจุบันจาก manifest ที่เผยแพร่ของ origin-server:

```bash
MANIFEST_URL=$(curl -sf https://api.github.com/repos/f5xc-salesdemos/origin-server/releases/latest \
  | python3 -c "import sys,json; assets=json.load(sys.stdin).get('assets',[]); print(next((a['browser_download_url'] for a in assets if a['name']=='manifest.json'),''))")
curl -sf "$MANIFEST_URL" | python3 -m json.tool
```

หากยังไม่มี release ให้ใช้แหล่งข้อมูลจาก repository โดยตรง:

```bash
curl -sf https://raw.githubusercontent.com/f5xc-salesdemos/origin-server/main/manifest.json | python3 -m json.tool
```

manifest จะแสดงรายการเส้นทางแอปพลิเคชันทั้งหมด, health check, container image และการจับคู่ฟีเจอร์สาธิต

### เส้นทางแอปพลิเคชัน

| เส้นทาง | แอปพลิเคชัน | ฟีเจอร์สาธิต |
|---|---|---|
| `/` | หน้า Landing | — |
| `/health` | Health check | — |
| `/juice-shop/` | OWASP Juice Shop | WAF, Bot Defense, API Security |
| `/dvwa/` | DVWA | WAF, Bot Defense |

| `/csd-demo/` | CSD Demo | Client-Side Defense |

## ขั้นตอนที่ 1: การเชื่อมต่อโดยตรง (พื้นฐาน)

```
┌──────────┐     ┌──────────────────────┐     ┌─────────────────────┐
│  Client   │────▶│  CDN Edge (NGINX)    │────▶│  Origin Server      │
│           │     │  20.65.90.112        │     │  20.12.78.159       │
└──────────┘     │  67+ CDN headers     │     │  Juice Shop, DVWA,  │
                 │  Disk cache           │     │  VAmPI, httpbin,    │
                 └──────────────────────┘     │  whoami, CSD Demo   │
                                              └─────────────────────┘
```

### กำหนดค่า Origin

ตั้งค่า IP ของ origin server ในการกำหนดค่า NGINX ของ CDN edge:

```bash
ssh azureuser@<CDN_EDGE_IP>
sudo sed -i 's|proxy_pass .*;|proxy_pass http://<ORIGIN_IP>;|' /etc/nginx/conf.d/cdn-edge.conf
sudo rm -rf /var/cache/nginx/cdn/*
sudo nginx -t && sudo systemctl reload nginx
```

หรือตั้งค่าในขณะ deploy ด้วย Terraform ผ่าน `terraform.tfvars`:

```hcl
origin_server = "http://<ORIGIN_IP>"
```

### ตรวจสอบแอปพลิเคชันทั้งหมด

ทดสอบแต่ละแอปพลิเคชันของ origin ผ่าน CDN:

```bash
CDN=<CDN_EDGE_IP>

# Health check (CDN local)
curl -sf "http://$CDN/health" | python3 -m json.tool

# Landing page
curl -sf -o /dev/null -w "/ : HTTP %{http_code}\n" "http://$CDN/"

# Juice Shop
curl -sf -o /dev/null -w "/juice-shop/ : HTTP %{http_code}\n" "http://$CDN/juice-shop/"

# DVWA (follows redirect to login)
curl -sf -o /dev/null -w "/dvwa/ : HTTP %{http_code}\n" -L "http://$CDN/dvwa/"

# VAmPI API
curl -sf "http://$CDN/vampi/users/v1" | python3 -m json.tool | head -5

# httpbin headers (shows CDN headers in JSON)
curl -sf "http://$CDN/httpbin/headers" | python3 -m json.tool | head -10

# whoami (shows ALL headers the origin receives)
curl -sf "http://$CDN/whoami/"

# CSD Demo
curl -sf -o /dev/null -w "/csd-demo/ : HTTP %{http_code}\n" "http://$CDN/csd-demo/"
```

เส้นทางทั้งหมดควรส่งคืน HTTP 200 (DVWA ส่งคืน 302 จากนั้น 200 เมื่อติดตาม redirect)

### ตรวจสอบ CDN Header ที่ Origin

endpoint `/whoami/` จะแสดง header ทุกรายการที่ origin ได้รับ เมื่อเข้าถึงผ่าน CDN จะแสดง vendor header ทั้ง 67+ รายการ:

```bash
curl -sf "http://$CDN/whoami/"
```

ตรวจสอบว่า header สำคัญเหล่านี้มีอยู่:

| ผู้ให้บริการ | Header | ค่าที่คาดหวัง |
|---|---|---|
| มาตรฐาน | `X-Forwarded-For` | `<your_ip>, <cdn_edge_ip>` |
| Akamai | `True-Client-Ip` | `<your_ip>` |
| Cloudflare | `Cf-Connecting-Ip` | `<your_ip>` |
| CloudFront | `Cloudfront-Viewer-Country` | `US` |
| Fastly | `Fastly-Client-Ip` | `<your_ip>` |
| Azure FD | `X-Azure-Clientip` | `<your_ip>` |

### เปรียบเทียบ Log

เปรียบเทียบ access log บนเซิร์ฟเวอร์ทั้งสองเครื่องเพื่อตรวจสอบการไหลของทราฟฟิก:

```bash
# CDN edge log — shows your client IP as source
ssh azureuser@<CDN_EDGE_IP> "sudo tail -5 /var/log/nginx/access.log"

# Origin log — shows CDN edge IP as source
ssh azureuser@<ORIGIN_IP> "sudo tail -5 /var/log/nginx/access.log"
```

log ของ origin ควรแสดง IP ของ CDN edge เป็นไคลเอนต์ที่เชื่อมต่อ ในขณะที่ IP จริงของไคลเอนต์จะถูกส่งผ่าน `X-Forwarded-For` และ header เฉพาะผู้ให้บริการ

### พฤติกรรมแคช

```bash
# First request — MISS (fetched from origin)
curl -s -I "http://$CDN/whoami/" | grep X-Cache-Status

# Second request — HIT (served from CDN cache)
curl -s -I "http://$CDN/whoami/" | grep X-Cache-Status
```

## ขั้นตอนที่ 2: การแทรก F5 XC (การสาธิตด้านความปลอดภัย)

หลังจากทดสอบพื้นฐานแล้ว ให้แทรก F5 XC HTTP load balancer ระหว่าง CDN และ origin:

```
┌──────────┐     ┌────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│  Client   │────▶│  CDN Edge      │────▶│  F5 XC HTTP LB   │────▶│  Origin Server  │
│           │     │  (NGINX)       │     │  WAF + Bot + API  │     │                 │
└──────────┘     └────────────────┘     └──────────────────┘     └─────────────────┘
```

1. **สร้าง F5 XC HTTP load balancer** โดยมี origin server ใน origin pool
2. **อัปเดต CDN edge** ให้ชี้ไปที่ F5 XC VIP แทนที่จะชี้ไปที่ origin โดยตรง:

```bash
ssh azureuser@<CDN_EDGE_IP>
sudo sed -i 's|proxy_pass .*;|proxy_pass https://<F5XC_LB_VIP>;|' /etc/nginx/conf.d/cdn-edge.conf
sudo rm -rf /var/cache/nginx/cdn/*
sudo nginx -t && sudo systemctl reload nginx
```

3. **ตรวจสอบการบังคับใช้ WAF** ตลอดทั้งห่วงโซ่:

```bash
# SQL injection through CDN → F5 XC WAF should block
curl -I "http://$CDN/dvwa/vulnerabilities/sqli/?id=%27+OR+1%3D1--"

# Normal request should pass
curl -sf -o /dev/null -w "%{http_code}" "http://$CDN/juice-shop/"
```

4. **กำหนดค่า Trusted Client IP Header** ใน F5 XC เพื่ออ่าน IP จริงของไคลเอนต์จาก CDN header (เช่น `True-Client-IP` สำหรับการจำลอง Akamai, `CF-Connecting-IP` สำหรับการจำลอง Cloudflare)

## สถาปัตยกรรมแบบหลายคอมโพเนนต์

| คอมโพเนนต์ | Repository | วัตถุประสงค์ |
|---|---|---|
| **CDN Edge** (นี้) | [cdn-simulator](https://github.com/f5xc-salesdemos/cdn-simulator) | แคช, vendor header |
| **Origin Server** | [origin-server](https://github.com/f5xc-salesdemos/origin-server) | เว็บแอปที่มีช่องโหว่ |
| **F5 XC Config** | หลายรายการ (waf, api-protection, bot-*, เป็นต้น) | นโยบายความปลอดภัย |

แต่ละคอมโพเนนต์เผยแพร่เอกสารที่ AI assistant อ่านเพื่อ deploy โครงสร้างพื้นฐาน origin-server เผยแพร่ [endpoint manifest](https://github.com/f5xc-salesdemos/origin-server/releases/latest) เป็น GitHub Release artifact ที่แสดงรายการเส้นทางแอปพลิเคชันและ health check ทั้งหมด
