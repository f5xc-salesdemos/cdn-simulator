## 제공 기능

- **[단일 엣지 노드](01-overview/)** — NGINX를 캐싱 리버스 프록시로 실행하는 Ubuntu 24.04 VM 1대, 최소 기능 CDN 엣지
- **[Terraform 배포](03-deploy/)** — 완전한 Azure 코드형 인프라: VNet, PIP, NSG, 그리고 cloud-init 프로비저닝이 포함된 VM
- **[67개 이상의 CDN 벤더 헤더](04-nginx-config/)** — Akamai, Cloudflare, CloudFront, Fastly, Azure Front Door의 요청 헤더를 시뮬레이션하며 True-Client-IP, CF-Connecting-IP, 지리적 위치, 디바이스 감지, JA3/JA4 핑거프린트, 봇 점수 포함
- **[오리진 통합](06-integrate/)** — 캐시 미스를 오리진으로서 F5 XC HTTP 로드 밸런서로 전달하도록 엣지 구성
- **[캐시 검증](05-verify/)** — X-Cache-Status 헤더와 NGINX 캐시 검사를 통해 HIT/MISS 동작 검증

## 빠른 참조

배포 후 CDN 엣지 노드는 다음을 노출합니다:

| 엔드포인트 | 메서드 | 용도 |
|---|---|---|
| `http://<PUBLIC_IP>/health` | GET | 헬스 체크 — 컴포넌트 상태와 벤더 프로필이 포함된 JSON 반환 |
| `http://<PUBLIC_IP>/` | GET | CDN 프록시 — 요청을 캐시하고 오리진 서버로 전달 |
| `http://<PUBLIC_IP>/<any-path>` | GET | CDN 프록시 — 모든 경로가 캐싱 및 벤더 헤더와 함께 프록시됨 |

모든 프록시 응답에 추가되는 응답 헤더:

| 헤더 | 예시 | 용도 |
|---|---|---|
| `X-Cache-Status` | `HIT` 또는 `MISS` | 응답이 캐시에서 제공되었는지 여부 |
| `X-CDN-Edge` | `cdn-simulator` | 이 엣지 노드를 식별 |
| `X-CDN-POP` | `SJC` | 시뮬레이션된 접속 지점(Point of Presence) 코드 |
| `X-Served-By` | `cache-sjc3120-SJC` | 시뮬레이션된 캐시 노드 식별자 |
| `X-Request-ID` | UUID | 요청별 고유 식별자 |
