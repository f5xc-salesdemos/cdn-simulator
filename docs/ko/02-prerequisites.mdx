## 필수 항목

### Azure 구독

다음을 생성할 수 있는 권한이 포함된 활성 Azure 구독이 필요합니다:

- 리소스 그룹
- 가상 네트워크 및 서브넷
- 네트워크 보안 그룹
- 공용 IP 주소
- 가상 머신

### Azure CLI

Azure CLI를 설치하고 인증합니다:

```bash
# Install (if not present)
curl -sL https://aka.ms/InstallAzureCLIDeb | sudo bash

# Authenticate
az login

# Verify subscription
az account show --query '{name:name, id:id}' -o table
```

### Terraform

Terraform >= 1.5가 필요합니다:

```bash
# Verify installation
terraform version

# If not installed, use HashiCorp's APT repository
wget -O- https://apt.releases.hashicorp.com/gpg | sudo gpg --dearmor -o /usr/share/keyrings/hashicorp-archive-keyring.gpg
echo "deb [signed-by=/usr/share/keyrings/hashicorp-archive-keyring.gpg] https://apt.releases.hashicorp.com $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/hashicorp.list
sudo apt update && sudo apt install terraform
```

### SSH 키 쌍

VM 접근을 위한 SSH 키 쌍이 필요합니다:

```bash
# Generate if you don't have one
ssh-keygen -t ed25519 -f ~/.ssh/cdn-edge-key -N ""
```

### Azure AD 권한

Azure AD 계정에 `User.Read` 권한이 있어야 합니다. 배포자 식별자(리소스 명명에 사용됨)는 Azure AD 프로필에서 자동으로 파생됩니다. 서비스 주체 또는 관리 ID 인증의 경우 `deployer` Terraform 변수를 명시적으로 설정하십시오.

## 선택 항목

### 사용자 지정 도메인

자체 서명된 인증서 대신 실제 TLS 인증서를 사용하려는 경우:

- DNS 제어가 가능한 도메인 이름
- Azure 공용 IP를 가리키는 A 레코드를 생성할 수 있는 권한

### 오리진 서버 URL

이 엣지 노드가 트래픽을 전달할 F5 XC HTTP 로드 밸런서의 URL입니다. 이 설정은 배포 후에도 구성할 수 있습니다 — 기본 NGINX 구성은 NGINX 설정에서 업데이트하는 플레이스홀더를 사용합니다.
