## O Que Este Projeto Oferece

- **[Nó de Borda Único](01-overview/)** — Uma VM Ubuntu 24.04 executando NGINX como proxy reverso de cache, o CDN de borda mínimo viável
- **[Implantação com Terraform](03-deploy/)** — Infraestrutura Azure completa como código: VNet, PIP, NSG e VM com provisionamento cloud-init
- **[Mais de 67 Cabeçalhos de Fornecedores CDN](04-nginx-config/)** — Simula cabeçalhos de requisição do Akamai, Cloudflare, CloudFront, Fastly e Azure Front Door, incluindo True-Client-IP, CF-Connecting-IP, geolocalização, detecção de dispositivo, impressões digitais JA3/JA4 e pontuações de bot
- **[Integração com Origem](06-integrate/)** — Configure o nó de borda para encaminhar falhas de cache a um balanceador de carga HTTP do F5 XC como origem
- **[Verificação de Cache](05-verify/)** — Valide o comportamento HIT/MISS com cabeçalhos X-Cache-Status e inspeção de cache do NGINX

## Referência Rápida

Após a implantação, o nó de borda CDN expõe:

| Endpoint | Método | Finalidade |
|---|---|---|
| `http://<PUBLIC_IP>/health` | GET | Verificação de integridade — retorna JSON com status dos componentes e perfis de fornecedores |
| `http://<PUBLIC_IP>/` | GET | Proxy CDN — armazena em cache e encaminha requisições ao servidor de origem |
| `http://<PUBLIC_IP>/<any-path>` | GET | Proxy CDN — todos os caminhos são proxiados com cache e cabeçalhos de fornecedor |

Cabeçalhos de resposta adicionados a cada resposta proxiada:

| Cabeçalho | Exemplo | Finalidade |
|---|---|---|
| `X-Cache-Status` | `HIT` ou `MISS` | Indica se a resposta foi servida a partir do cache |
| `X-CDN-Edge` | `cdn-simulator` | Identifica este nó de borda |
| `X-CDN-POP` | `SJC` | Código simulado de Ponto de Presença |
| `X-Served-By` | `cache-sjc3120-SJC` | Identificador simulado do nó de cache |
| `X-Request-ID` | UUID | Identificador único por requisição |
