## O Que Isto Oferece

- **[Nó de Borda Único](01-overview/)** — Uma VM Ubuntu 24.04 executando NGINX como proxy reverso com cache, o CDN edge mínimo viável
- **[Implantação com Terraform](03-deploy/)** — Infraestrutura completa como código no Azure: VNet, PIP, NSG e VM com provisionamento via cloud-init
- **[Mais de 67 Headers de Fornecedores CDN](04-nginx-config/)** — Simula headers de requisição da Akamai, Cloudflare, CloudFront, Fastly e Azure Front Door, incluindo True-Client-IP, CF-Connecting-IP, geolocalização, detecção de dispositivo, fingerprints JA3/JA4 e scores de bot
- **[Integração com Origem](06-integrate/)** — Configure o nó de borda para encaminhar cache misses para um HTTP load balancer do F5 XC como origem
- **[Verificação de Cache](05-verify/)** — Valide o comportamento de HIT/MISS com headers X-Cache-Status e inspeção do cache do NGINX

## Referência Rápida

Após a implantação, o nó de borda do CDN expõe:

| Endpoint | Método | Finalidade |
|---|---|---|
| `http://<PUBLIC_IP>/health` | GET | Verificação de saúde — retorna JSON com status dos componentes e perfis de fornecedores |
| `http://<PUBLIC_IP>/` | GET | Proxy CDN — armazena em cache e encaminha requisições para o servidor de origem |
| `http://<PUBLIC_IP>/<any-path>` | GET | Proxy CDN — todos os caminhos são proxificados com cache e headers de fornecedores |

Headers de resposta adicionados a cada resposta proxificada:

| Header | Exemplo | Finalidade |
|---|---|---|
| `X-Cache-Status` | `HIT` ou `MISS` | Indica se a resposta foi servida a partir do cache |
| `X-CDN-Edge` | `cdn-simulator` | Identifica este nó de borda |
| `X-CDN-POP` | `SJC` | Código simulado de Ponto de Presença |
| `X-Served-By` | `cache-sjc3120-SJC` | Identificador simulado do nó de cache |
| `X-Request-ID` | UUID | Identificador único por requisição |
