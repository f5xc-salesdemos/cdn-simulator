## 提供する機能

- **[単一エッジノード](01-overview/)** — NGINX をキャッシング リバースプロキシとして実行する Ubuntu 24.04 VM 1台、最小構成の CDN エッジ
- **[Terraform デプロイメント](03-deploy/)** — 完全な Azure Infrastructure as Code: VNet、PIP、NSG、cloud-init プロビジョニング付き VM
- **[67 以上の CDN ベンダーヘッダー](04-nginx-config/)** — Akamai、Cloudflare、CloudFront、Fastly、Azure Front Door のリクエストヘッダーをシミュレーション（True-Client-IP、CF-Connecting-IP、ジオロケーション、デバイス検出、JA3/JA4 フィンガープリント、ボットスコアを含む）
- **[オリジン統合](06-integrate/)** — キャッシュミスを F5 XC HTTP ロードバランサーにオリジンとして転送するようエッジを設定
- **[キャッシュ検証](05-verify/)** — X-Cache-Status ヘッダーと NGINX キャッシュ検査による HIT/MISS 動作の検証

## クイックリファレンス

デプロイ後、CDN エッジノードは以下のエンドポイントを公開します:

| エンドポイント | メソッド | 目的 |
|---|---|---|
| `http://<PUBLIC_IP>/health` | GET | ヘルスチェック — コンポーネントステータスとベンダープロファイルを含む JSON を返す |
| `http://<PUBLIC_IP>/` | GET | CDN プロキシ — リクエストをキャッシュしてオリジンサーバーに転送 |
| `http://<PUBLIC_IP>/<any-path>` | GET | CDN プロキシ — すべてのパスがキャッシングとベンダーヘッダー付きでプロキシされる |

すべてのプロキシレスポンスに追加されるレスポンスヘッダー:

| ヘッダー | 例 | 目的 |
|---|---|---|
| `X-Cache-Status` | `HIT` または `MISS` | レスポンスがキャッシュから配信されたかどうか |
| `X-CDN-Edge` | `cdn-simulator` | このエッジノードを識別 |
| `X-CDN-POP` | `SJC` | シミュレートされた Point of Presence コード |
| `X-Served-By` | `cache-sjc3120-SJC` | シミュレートされたキャッシュノード識別子 |
| `X-Request-ID` | UUID | リクエストごとの一意の識別子 |
