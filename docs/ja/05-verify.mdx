デプロイ後、CDN エッジノードが正しく機能していることを確認するために、以下のチェックを実行してください。`terraform apply` の後、cloud-init が NGINX のインストールを完了するまで 2〜3 分お待ちください。

## ヘルスチェック

NGINX が稼働しレスポンスを返していることを確認します：

```bash
curl -s "http://<PUBLIC_IP>/health" | jq .
```

期待される出力：

```json
{
  "status": "healthy",
  "component": "cdn-edge",
  "engine": "nginx",
  "vendor_profiles": [
    "akamai",
    "cloudflare",
    "cloudfront",
    "fastly",
    "azure-front-door"
  ]
}
```

Terraform の出力を使用する場合：

```bash
curl -s "$(terraform output -raw health_check_url)" | jq .
```

## キャッシュミス（最初のリクエスト）

任意のパスへの最初のリクエストはキャッシュミスとなります — エッジがオリジンからフェッチします：

```bash
curl -I "http://<PUBLIC_IP>/"
```

レスポンスに以下のヘッダーがあることを確認します：

```
X-Cache-Status: MISS
X-CDN-Edge: cdn-simulator
```

`MISS` は、コンテンツがキャッシュになく、オリジンサーバーからフェッチされたことを意味します。

## キャッシュヒット（後続のリクエスト）

同じリクエストを直後に繰り返します：

```bash
curl -I "http://<PUBLIC_IP>/"
```

期待されるヘッダー：

```
X-Cache-Status: HIT
X-CDN-Edge: cdn-simulator
```

`HIT` は、オリジンに接続せずに NGINX ディスクキャッシュからレスポンスが配信されたことを確認します。

## キャッシュディレクトリの検査

VM に SSH 接続して、キャッシュされたコンテンツがディスク上に存在することを確認します：

```bash
ssh azureuser@<PUBLIC_IP>

# キャッシュディレクトリにコンテンツがあることを確認
sudo find /var/cache/nginx/cdn -type f | head -20

# キャッシュサイズを確認
sudo du -sh /var/cache/nginx/cdn
```

## NGINX ステータス

NGINX が稼働しており、設定が有効であることを確認します：

```bash
ssh azureuser@<PUBLIC_IP>

# サービスステータスを確認
sudo systemctl status nginx

# 設定を検証
sudo nginx -t

# アクティブな接続を表示（stub_status が有効な場合）
curl -s http://localhost/health
```

## オリジン接続性

エッジノードがオリジンサーバーに到達できることを確認します：

```bash
ssh azureuser@<PUBLIC_IP>

# オリジンへの接続をテスト（お使いのオリジン URL に置き換えてください）
curl -I "https://your-origin.example.com/"
```

これが失敗する場合は、以下を確認してください：

1. Terraform 設定で `origin_server` 変数が正しいこと
2. オリジンサーバーがエッジノードのパブリック IP からの受信接続を許可していること
3. VM から DNS 解決が機能すること（`nslookup your-origin.example.com`）

## Cloud-Init の進行状況

cloud-init ログでプロビジョニングの進行状況を監視します：

```bash
ssh azureuser@<PUBLIC_IP> "tail -f /var/log/cloud-init-progress.log"
```

期待されるフェーズ：`[init]`、`[nic]`（動的 NIC 検出）、`[complete]`。

cloud-init がエラーを報告する場合：

```bash
ssh azureuser@<PUBLIC_IP> "sudo cloud-init status --long"
```

## エンドツーエンドテスト

完全なリクエストサイクルを実行し、レスポンスチェーンを検証します：

```bash
# 最初のリクエスト — MISS（オリジンからフェッチ）
echo "=== Request 1 (expect MISS) ==="
curl -s -o /dev/null -w "HTTP %{http_code} | Cache: %{header:X-Cache-Status}\n" "http://<PUBLIC_IP>/test-path"

# 2回目のリクエスト — HIT（キャッシュから配信）
echo "=== Request 2 (expect HIT) ==="
curl -s -o /dev/null -w "HTTP %{http_code} | Cache: %{header:X-Cache-Status}\n" "http://<PUBLIC_IP>/test-path"
```

期待される出力：

```
=== Request 1 (expect MISS) ===
HTTP 200 | Cache: MISS
=== Request 2 (expect HIT) ===
HTTP 200 | Cache: HIT
```

## ベンダーヘッダーの検証

すべての CDN ベンダーヘッダーが注入されていることを確認するには、`/headers` パスにリクエストします（httpbin.org をオリジンとして使用している場合）：

```bash
curl -s "http://<PUBLIC_IP>/headers" | python3 -m json.tool
```

期待されるレスポンスには 5 つのベンダーすべてのヘッダーが含まれます：

```json
{
    "headers": {
        "True-Client-Ip": "<YOUR_CLIENT_IP>",
        "Cf-Connecting-Ip": "<YOUR_CLIENT_IP>",
        "Cf-Ipcountry": "US",
        "Cf-Ray": "<request_id>-SJC",
        "Cf-Bot-Score": "85",
        "Cf-Ja3-Hash": "e7d705a3286e19ea42f587b344ee6865",
        "Cloudfront-Viewer-Country": "US",
        "Cloudfront-Viewer-City": "San Jose",
        "Cloudfront-Is-Desktop-Viewer": "true",
        "Cloudfront-Is-Mobile-Viewer": "false",
        "Fastly-Client-Ip": "<YOUR_CLIENT_IP>",
        "X-Akamai-Edgescape": "georegion=263,country_code=US,...",
        "X-Azure-Clientip": "<YOUR_CLIENT_IP>",
        "X-Azure-Fdid": "a0a0a0a0-bbbb-cccc-dddd-e1e1e1e1e1e1",
        "X-Geo-Country-Code": "US"
    }
}
```

### デバイス検出テスト

モバイル User-Agent でテストして、デバイス検出ヘッダーが変化することを確認します：

```bash
curl -s "http://<PUBLIC_IP>/headers?device=mobile" \
  -H "User-Agent: Mozilla/5.0 (iPhone; CPU iPhone OS 17_0 like Mac OS X)" \
  | python3 -m json.tool | grep -E "Is-Mobile|Is-Desktop|is_mobile"
```

期待される結果：

```
"Cloudfront-Is-Desktop-Viewer": "false",
"Cloudfront-Is-Mobile-Viewer": "true",
```

## トラブルシューティング

### NGINX が稼働していない

```bash
ssh azureuser@<PUBLIC_IP>
sudo systemctl status nginx
sudo journalctl -u nginx --no-pager -n 50
```

### Cloud-init が完了していない

`terraform apply` 後、cloud-init には 2〜3 分かかります。進行状況を確認します：

```bash
ssh azureuser@<PUBLIC_IP>
sudo cloud-init status
sudo tail -f /var/log/cloud-init-output.log
```

### キャッシュが常に MISS を表示する

- オリジンがキャッシュ可能なレスポンスコード（200、301、302）を返していることを確認してください
- オリジンが `Cache-Control: no-cache` または `no-store` ヘッダーを送信していないことを確認してください
- NGINX エラーログを確認してください：`sudo tail -f /var/log/nginx/error.log`

### ポート 80/443 で接続が拒否される

- NSG ルールが適用されていることを確認してください：`az network nsg rule list --nsg-name "$(terraform output -raw nsg_name)" --resource-group "$(terraform output -raw resource_group_name)" -o table`
- NGINX がリッスンしていることを確認してください：`ssh azureuser@<PUBLIC_IP> "sudo ss -tlnp | grep nginx"`
