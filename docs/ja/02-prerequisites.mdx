## 必須

### Azureサブスクリプション

以下を作成する権限を持つ有効なAzureサブスクリプション：

- リソースグループ
- 仮想ネットワークとサブネット
- ネットワークセキュリティグループ
- パブリックIPアドレス
- 仮想マシン

### Azure CLI

Azure CLIをインストールして認証します：

```bash
# Install (if not present)
curl -sL https://aka.ms/InstallAzureCLIDeb | sudo bash

# Authenticate
az login

# Verify subscription
az account show --query '{name:name, id:id}' -o table
```

### Terraform

Terraform >= 1.5 が必要です：

```bash
# Verify installation
terraform version

# If not installed, use HashiCorp's APT repository
wget -O- https://apt.releases.hashicorp.com/gpg | sudo gpg --dearmor -o /usr/share/keyrings/hashicorp-archive-keyring.gpg
echo "deb [signed-by=/usr/share/keyrings/hashicorp-archive-keyring.gpg] https://apt.releases.hashicorp.com $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/hashicorp.list
sudo apt update && sudo apt install terraform
```

### SSHキーペア

VMアクセス用のSSHキーペア：

```bash
# Generate if you don't have one
ssh-keygen -t ed25519 -f ~/.ssh/cdn-edge-key -N ""
```

### Azure AD権限

Azure ADアカウントには `User.Read` 権限が必要です。デプロイヤー識別子（リソース命名に使用）は、Azure ADプロファイルから自動的に取得されます。サービスプリンシパルまたはマネージドID認証の場合は、`deployer` Terraform変数を明示的に設定してください。

## オプション

### カスタムドメイン

自己署名証明書の代わりに実際のTLS証明書を使用したい場合：

- DNS管理権限のあるドメイン名
- AzureパブリックIPを指すAレコードを作成する機能

### オリジンサーバーURL

このエッジノードがトラフィックを転送するF5 XC HTTPロードバランサーのURL。これはデプロイ後に設定可能です — デフォルトのNGINX設定ではプレースホルダーが使用されており、NGINX設定で更新します。
