## 本專案提供的功能

- **[單一邊緣節點](01-overview/)** — 一台執行 NGINX 作為快取反向代理的 Ubuntu 24.04 虛擬機，最小可行的 CDN 邊緣節點
- **[Terraform 部署](03-deploy/)** — 完整的 Azure 基礎架構即程式碼：VNet、PIP、NSG，以及使用 cloud-init 佈建的虛擬機
- **[67+ CDN 供應商標頭](04-nginx-config/)** — 模擬 Akamai、Cloudflare、CloudFront、Fastly 和 Azure Front Door 的請求標頭，包括 True-Client-IP、CF-Connecting-IP、地理位置、裝置偵測、JA3/JA4 指紋及機器人評分
- **[來源站整合](06-integrate/)** — 設定邊緣節點將快取未命中的請求轉發至 F5 XC HTTP 負載平衡器作為來源站
- **[快取驗證](05-verify/)** — 透過 X-Cache-Status 標頭和 NGINX 快取檢查驗證 HIT/MISS 行為

## 快速參考

部署完成後，CDN 邊緣節點公開以下端點：

| 端點 | 方法 | 用途 |
|---|---|---|
| `http://<PUBLIC_IP>/health` | GET | 健康檢查 — 回傳包含元件狀態和供應商設定檔的 JSON |
| `http://<PUBLIC_IP>/` | GET | CDN 代理 — 快取並轉發請求至來源伺服器 |
| `http://<PUBLIC_IP>/<any-path>` | GET | CDN 代理 — 所有路徑皆會透過快取和供應商標頭進行代理 |

每個代理回應中新增的回應標頭：

| 標頭 | 範例 | 用途 |
|---|---|---|
| `X-Cache-Status` | `HIT` 或 `MISS` | 回應是否從快取提供 |
| `X-CDN-Edge` | `cdn-simulator` | 識別此邊緣節點 |
| `X-CDN-POP` | `SJC` | 模擬的接入點代碼 |
| `X-Served-By` | `cache-sjc3120-SJC` | 模擬的快取節點識別碼 |
| `X-Request-ID` | UUID | 每個請求的唯一識別碼 |
