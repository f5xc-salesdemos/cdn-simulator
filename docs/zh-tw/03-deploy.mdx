所有 Terraform 檔案均位於 [`terraform/`](https://github.com/f5xc-salesdemos/cdn-simulator/tree/main/terraform) 目錄中。複製儲存庫並直接部署：

```bash
git clone https://github.com/f5xc-salesdemos/cdn-simulator.git
cd cdn-simulator/terraform
cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars with your Azure subscription ID and origin server
```

## Terraform 設定

### Terraform 檔案結構

terraform 目錄包含 9 個遵循示範資源標準的檔案：

- `versions.tf` — Terraform 及提供者版本限制（azurerm ~> 4.0、azuread ~> 3.0）
- `providers.tf` — Azure RM 及 Azure AD 提供者設定
- `data.tf` — 用於部署者自動解析的 Azure AD 資料來源
- `locals.tf` — 部署者解析、Azure 雲端採用框架資源命名、標準標籤
- `main.tf` — 資源群組（命名為 `rg-cdn-simulator-{environment}-{deployer}`）
- `variables.tf` — 所有輸入變數（3 個必要、8 個選用）
- `network.tf` — VNet（10.100.0.0/16）、子網路、公用 IP、NSG（連接埠 22/80/443）、NIC
- `vm.tf` — 搭載透過 templatefile() 傳入 cloud-init 的 Ubuntu 24.04 VM
- `outputs.tf` — 17 個輸出（15 個標準 + 2 個元件專用）

`variables.tf` 定義了 11 個輸入變數，分為「一般」、「運算」及「元件專用」三個區段。`deployer` 識別碼會從您的 Azure AD 帳戶自動解析 — 您只需設定 `subscription_id`、`origin_server` 及 `origin_host`：

```hcl file=./_data/variables.tf
```

### 網路基礎架構

`network.tf` 建立 VNet、子網路、公用 IP、NSG（連接埠 22/80/443）及 NIC：

```hcl file=./_data/network.tf
```

### 搭載 Cloud-Init 的虛擬機器

`vm.tf` 建立 Ubuntu 24.04 VM。SSH 公開金鑰路徑透過 `pathexpand()` 展開，以處理 `~`。Cloud-init 範本接收 `origin_server` 及 `origin_host` 變數：

```hcl file=./_data/vm.tf
```

### Cloud-Init 佈建

`cloud-init.yaml` 使用核心調校、systemd 限制、效能最佳化設定的 NGINX、128 MB 快取鍵值區域、上游保持連線池、gzip 壓縮以及 67 種以上 CDN 廠商標頭來佈建 VM。共用輔助程式庫（`/usr/local/lib/cloud-init-helpers.sh`）提供重試邏輯，並將進度記錄至 `/var/log/cloud-init-progress.log`。

Cloud-init 使用 Terraform 範本變數：`${origin_server}` 及 `${origin_host}` 用於上游設定。NGINX 變數（例如 `${request_id}`）在 Terraform templatefile 中以 `$${request_id}` 的形式跳脫。

```yaml file=./_data/cloud-init.yaml
```

### 輸出

`outputs.tf` 依循示範資源標準公開 17 個輸出 — 15 個所有示範資源共用的標準輸出（`deployer`、`public_ip`、`private_ip`、`ssh_command`、`resource_group_name`、`vm_name`、`nsg_name`、`vnet_name`、`subnet_id`、`component`、`environment`、`resource_group_id`、`vm_id`、`nsg_id`、`location`），以及 2 個元件專用輸出（`edge_url`、`health_check_url`）：

```hcl file=./_data/outputs.tf
```

### 範例變數檔案

將 `terraform.tfvars.example` 複製為 `terraform.tfvars` 並填入您的值。`.gitignore` 已排除 `terraform.tfvars`，以防止提交認證資訊：

```hcl file=./_data/terraform.tfvars.example
```

## 部署

```bash
# Initialize Terraform
terraform init

# Review the plan
terraform plan

# Apply
terraform apply
```

成功部署後，Terraform 會輸出公用 IP、SSH 指令及邊緣節點 URL。

## VM 規格選擇

F 系列運算最佳化 VM 適合用於此以 CPU 為瓶頸的 NGINX 代理工作負載。預設的 `Standard_F4s_v2`（4 vCPU、8 GiB）適合實驗室及示範用途。若要進行負載測試或正式環境效能基準測試，可覆寫 `vm_size` 變數。Cloud-init 核心調校及 NGINX 設定會依 vCPU 數量自動擴展（`worker_processes auto`）。

## 部署後步驟

`terraform apply` 完成後，請等待 2 至 3 分鐘讓 cloud-init 完成 NGINX 的安裝與設定。驗證健康狀態端點：

```bash
curl -s "http://$(terraform output -raw public_ip)/health" | jq .
```

預期回應：

```json
{
  "status": "healthy",
  "component": "cdn-edge",
  "engine": "nginx",
  "vendor_profiles": [
    "akamai",
    "cloudflare",
    "cloudfront",
    "fastly",
    "azure-front-door"
  ]
}
```

## 與上游元件串接

CDN 模擬器需要已部署的來源伺服器。使用來源伺服器的 terraform 輸出來填入必要變數：

```bash
cd ../origin-server/terraform
origin_ip=$(terraform output -raw public_ip)

cd ../../cdn-simulator/terraform
cat > terraform.tfvars <<EOF
subscription_id = "your-subscription-id"
origin_server   = "http://${origin_ip}"
origin_host     = "${origin_ip}:80"
EOF
```

| 必要變數 | 來源 | 格式 |
|---|---|---|
| `origin_server` | 來源伺服器 `public_ip` 輸出 | `http://<ip>`（包含協定） |
| `origin_host` | 來源伺服器 `public_ip` 輸出 | `<ip>:80`（不含協定，包含連接埠） |

繼續前往 [NGINX 設定](../04-nginx-config/) 了解自訂選項，或前往 [驗證](../05-verify/) 進行快取測試。
