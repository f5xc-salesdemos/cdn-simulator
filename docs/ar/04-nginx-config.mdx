يقوم تجهيز cloud-init بنشر تكوين حافة CDN محسّن بالكامل للأداء على NGINX. توثّق هذه الصفحة كل طبقة من طبقات التكوين، من ضبط النواة مرورًا بسلوك التخزين المؤقت وصولًا إلى حقن ترويسات المزودين. تم التحقق من جميع الإعدادات خلال اختبار حمل مستمر لمدة 48 ساعة بذروة بلغت 172,540 طلب/ثانية.

## ملفات التكوين

| الملف | الغرض |
|---|---|
| `/etc/sysctl.d/99-cdn-tuning.conf` | ضبط شبكة نواة Linux |
| `/etc/systemd/system/nginx.service.d/override.conf` | حدود واصفات الملفات لـ NGINX |
| `/etc/security/limits.d/99-nginx.conf` | حدود مستوى نظام التشغيل لمستخدم www-data |
| `/etc/nginx/nginx.conf` | تكوين NGINX الرئيسي (العمال، المخازن المؤقتة، gzip، التسجيل) |
| `/etc/nginx/conf.d/cdn-edge.conf` | تكوين وكيل CDN (التخزين المؤقت، الخادم الأصلي، الترويسات) |

## تكوين NGINX الرئيسي

### العمال والاتصالات

```nginx
worker_processes auto;           # 4 workers on D4s_v5 (1 per vCPU)
worker_rlimit_nofile 65535;      # Per-worker file descriptor limit

events {
    use epoll;                   # Linux-optimized event model
    worker_connections 8192;     # 4 workers x 8192 = 32,768 max concurrent
    multi_accept on;             # Accept all pending connections per event loop
    accept_mutex off;            # Not needed with epoll + reuseport
}
```

يضبط تجاوز Systemd في `/etc/systemd/system/nginx.service.d/override.conf` قيمة `LimitNOFILE=65535` لتتوافق.

### مخازن الوكيل المؤقتة

```nginx
proxy_buffering on;
proxy_buffer_size 16k;           # Header buffer (handles large CDN headers)
proxy_buffers 64 16k;            # 1 MB per connection (64 x 16k)
proxy_busy_buffers_size 256k;    # Can send 256k to client while still reading
```

حجم المخزن المؤقت المشغول البالغ 256k أمر حاسم — يجب أن يتجاوز أكبر استجابة مخزنة مؤقتًا (Juice Shop بحجم 75 كيلوبايت). تسبب الإعداد الأصلي البالغ 64k في تسلسل تحت الحمل.

### ضغط Gzip

```nginx
gzip on;
gzip_comp_level 4;               # Balance: ~85% of max compression at ~40% CPU
gzip_min_length 256;             # Skip tiny responses (gzip header overhead)
gzip_vary on;                    # Vary: Accept-Encoding for correct caching
gzip_proxied any;                # Compress all proxied responses
gzip_types text/plain text/css text/javascript text/xml
           application/json application/javascript application/xml
           application/xml+rss application/atom+xml
           application/ld+json application/manifest+json
           image/svg+xml;
```

### تخزين الملفات المفتوحة مؤقتًا

```nginx
open_file_cache max=200000 inactive=20s;
open_file_cache_valid 30s;
open_file_cache_min_uses 2;
open_file_cache_errors on;
```

يخزّن واصفات الملفات والبيانات الوصفية للكائنات المخزنة مؤقتًا، مما يلغي استدعاءات النظام `stat()` و `open()` على الملفات المتكررة الوصول.

### Keepalive العميل

```nginx
keepalive_timeout 65;
keepalive_requests 100000;       # Was 1000 — caused connection recycling at 90K req/s
```

عند 90K طلب/ثانية مع `keepalive_requests 1000`، كانت الاتصالات تُعاد تدويرها كل 11 ثانية، مما يولّد مقابس TIME_WAIT. أدت زيادة القيمة إلى 100,000 إلى القضاء على هذه المشكلة تمامًا.

### تسجيل الوصول

```nginx
log_format cdn '$remote_addr [$time_local] "$request" $status $body_bytes_sent $upstream_cache_status $request_time';
access_log /var/log/nginx/access.log cdn buffer=256k flush=5s;
```

يتضمن تنسيق سجل `cdn` المتغير `$upstream_cache_status` (HIT/MISS) و `$request_time` لتحليل زمن الاستجابة. يقلل التسجيل المخزّن مؤقتًا (256k/تفريغ كل 5 ثوانٍ) من حمل الإدخال/الإخراج تحت الحمل العالي.

## Keepalive الخادم الأصلي

```nginx
upstream origin_backend {
    server 20.12.78.159:80;
    keepalive 256;               # Persistent connections per worker to origin
    keepalive_timeout 60s;
    keepalive_requests 1000;
}
```

مع 4 عمال × 256 keepalive = 1,024 اتصال نشط مع الخادم الأصلي. بالاقتران مع `proxy_http_version 1.1` و `proxy_set_header Connection ""`، يتم القضاء على حمل مصافحة TCP عند كل إخفاق في التخزين المؤقت.

## تكوين التخزين المؤقت

### مسار التخزين المؤقت

```nginx
proxy_cache_path /var/cache/nginx/cdn
                 levels=1:2
                 keys_zone=cdn_cache:32m
                 max_size=25g
                 inactive=24h
                 use_temp_path=off;
```

| المعامل | القيمة | الغرض |
|---|---|---|
| `keys_zone=cdn_cache:32m` | 32 ميجابايت ذاكرة مشتركة | يخزّن حوالي 256,000 مفتاح تخزين مؤقت وبيانات وصفية |
| `max_size=25g` | حد 25 جيجابايت على القرص | يستخدم معظم قرص نظام التشغيل بحجم 30 جيجابايت؛ إخلاء LRU عند الوصول للحد |
| `inactive=24h` | مهلة خمول 24 ساعة | يتم إخلاء المحتوى غير المُصل إليه لمدة 24 ساعة (ليس مثل TTL) |
| `use_temp_path=off` | الكتابة مباشرة إلى مجلد التخزين المؤقت | يتجنب نقل الملفات عبر أنظمة الملفات |

### سلوك التخزين المؤقت

```nginx
proxy_cache cdn_cache;
proxy_cache_valid 200 301 302 4h;
proxy_cache_valid 404 1m;
proxy_cache_key "$scheme$host$request_uri";
proxy_cache_lock on;
proxy_cache_lock_age 3s;
proxy_cache_lock_timeout 3s;
proxy_cache_background_update on;
proxy_cache_use_stale updating error timeout http_500 http_502 http_503 http_504;
proxy_ignore_headers Set-Cookie Cache-Control Expires Vary;

proxy_hide_header X-Cache-Status;
proxy_hide_header Vary;
```

| التوجيه | القيمة | الغرض |
|---|---|---|
| `proxy_cache_valid 200 301 302 4h` | TTL مدته 4 ساعات | المحتوى المخزّن صالح لمدة 4 ساعات (تم زيادته لتقليل انخفاضات الإنتاجية أثناء موجة التحديث) |
| `proxy_cache_lock on` | منع التدافع الجماعي | طلب واحد فقط يذهب للخادم الأصلي لكل عنوان URL غير مخزّن مؤقتًا؛ الباقي ينتظر ملء التخزين المؤقت |
| `proxy_cache_lock_age 3s` | مهلة القفل | إذا لم يكتمل الطلب الأول خلال 3 ثوانٍ، يُسمح بمرور طلب آخر |
| `proxy_cache_background_update on` | تحديث بدون زمن انتظار | يقدّم المحتوى القديم فورًا أثناء التحديث في الخلفية |
| `proxy_cache_use_stale` | المرونة | يقدّم المحتوى القديم أثناء أخطاء الخادم الأصلي (500/502/503/504) أو أثناء التحديث |
| `proxy_ignore_headers` | فرض التخزين المؤقت | يتجاهل ترويسات Set-Cookie وCache-Control وExpires وVary من الخادم الأصلي — شبكة CDN هي من تقرر TTL وسلوك Vary (يرسل Juice Shop `max-age=0` وترويسات `Vary` ثلاثية مما منع التخزين المؤقت الفعّال) |
| `proxy_hide_header X-Cache-Status` | إزالة ترويسة الخادم الأصلي | يضيف NGINX الأصلي ترويسة X-Cache-Status خاصة به — تُزال حتى تظهر فقط حالة تخزين CDN المؤقت |
| `proxy_hide_header Vary` | منع تجزئة التخزين المؤقت | يرسل الخادم الأصلي ترويسات `Vary: Accept-Encoding` متعددة. إزالتها تمنع تجزئة مفتاح التخزين المؤقت عبر تباديل `Accept-Encoding`. يضيف `gzip_vary on` في NGINX ترويسة `Vary` واحدة صحيحة تلقائيًا |

### مهل الوكيل

```nginx
proxy_read_timeout 30s;
proxy_connect_timeout 10s;
proxy_send_timeout 15s;
```

يمنح الخادم الأصلي وقتًا أكبر للاستجابة تحت الحمل مع الفشل السريع عند مشاكل الاتصال.

### كتلة الخادم

```nginx
server {
    listen 80 reuseport;        # Kernel distributes connections across all 4 workers
    server_name _;
}
```

يُفعّل `reuseport` خيار `SO_REUSEPORT` — حيث توزّع النواة الاتصالات الواردة مباشرة على عمليات العمال، مما يلغي تنافس قفل القبول.

## ترويسات مزودي CDN

يحقن المحاكي ترويسات من جميع مزودي CDN الخمسة الرئيسيين في وقت واحد. يسمح هذا بتكوين F5 XC مع "ترويسة IP العميل الموثوقة" لأي مزود ورؤية حمولات ترويسات واقعية بغض النظر عن شبكة CDN التي يتم محاكاتها.

## اكتشاف الجهاز

يكتشف المحاكي نوع الجهاز من ترويسة `User-Agent` باستخدام توجيهات `map` في NGINX:

- **الهاتف المحمول**: يطابق iPhone، Android (غير اللوحي)، iPod، BlackBerry، Opera Mini، IEMobile
- **الجهاز اللوحي**: يطابق iPad، أجهزة Android اللوحية، Kindle، PlayBook
- **سطح المكتب**: الافتراضي عندما لا يتطابق أي من الهاتف المحمول أو الجهاز اللوحي

ينعكس نوع الجهاز في:
- `CloudFront-Is-Desktop-Viewer` / `CloudFront-Is-Mobile-Viewer` / `CloudFront-Is-Tablet-Viewer`
- `X-Akamai-Device-Characteristics` (حقول `is_mobile` و`is_tablet` و`is_wireless_device`)

## العمليات التشغيلية

### تغيير الخادم الأصلي

```bash
ssh azureuser@<PUBLIC_IP>

# Update upstream server
sudo sed -i 's|server .*;|server NEW_HOST:80;|' /etc/nginx/conf.d/cdn-edge.conf

# Clear cache and reload
sudo rm -rf /var/cache/nginx/cdn/*
sudo nginx -t && sudo systemctl reload nginx
```

أو قم بتحديث `origin_host` في `terraform.tfvars` وتشغيل `terraform apply` لإعادة التجهيز.

### مسح التخزين المؤقت

```bash
ssh azureuser@<PUBLIC_IP>
sudo rm -rf /var/cache/nginx/cdn/*
sudo systemctl reload nginx
```

### فحص إحصائيات التخزين المؤقت

```bash
# Object count and disk usage
sudo find /var/cache/nginx/cdn -type f | wc -l
sudo du -sh /var/cache/nginx/cdn

# Recent access log with cache status
tail -20 /var/log/nginx/access.log
```

### المراقبة تحت الحمل

```bash
# Real-time connections and socket states
ss -s

# NGINX worker CPU usage
top -bn1 | grep nginx

# Upstream keepalive connections
ss -tn state established dst <ORIGIN_IP> | wc -l

# TIME_WAIT socket count
ss -tn state time-wait | wc -l
```
