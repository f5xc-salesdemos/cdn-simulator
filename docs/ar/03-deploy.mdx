جميع ملفات Terraform موجودة في مجلد [`terraform/`](https://github.com/f5xc-salesdemos/cdn-simulator/tree/main/terraform). انسخ المستودع وانشر مباشرةً:

```bash
git clone https://github.com/f5xc-salesdemos/cdn-simulator.git
cd cdn-simulator/terraform
cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars with your Azure subscription ID and origin server
```

## تكوين Terraform

### هيكل ملفات Terraform

يحتوي مجلد terraform على 9 ملفات تتبع معيار موارد العرض التوضيحي:

- `versions.tf` — قيود إصدار Terraform والموفر (azurerm ~> 4.0, azuread ~> 3.0)
- `providers.tf` — تكوين موفر Azure RM وAzure AD
- `data.tf` — مصادر بيانات Azure AD لحل المُنشِر تلقائياً
- `locals.tf` — حل المُنشِر، وتسمية الموارد وفق إطار Azure Cloud Adoption Framework، والعلامات القياسية
- `main.tf` — مجموعة الموارد (المسماة `rg-cdn-simulator-{environment}-{deployer}`)
- `variables.tf` — جميع متغيرات الإدخال (3 مطلوبة، 8 اختيارية)
- `network.tf` — الشبكة الافتراضية (10.100.0.0/16)، والشبكة الفرعية، والـ IP العام، ومجموعة أمان الشبكة (المنافذ 22/80/443)، وبطاقة الشبكة
- `vm.tf` — جهاز VM بنظام Ubuntu 24.04 مع cloud-init عبر templatefile()
- `outputs.tf` — 17 مخرجاً (15 قياسياً + 2 خاصاً بالمكوّن)

يُعرِّف `variables.tf` ما مجموعه 11 متغير إدخال مُنظَّمة في أقسام: عام، وحوسبة، وخاص بالمكوّن. يتم حل معرّف `deployer` تلقائياً من حسابك على Azure AD — تحتاج فقط إلى تعيين `subscription_id` و`origin_server` و`origin_host`:

```hcl file=./_data/variables.tf
```

### البنية التحتية للشبكات

يُنشئ `network.tf` الشبكة الافتراضية، والشبكة الفرعية، والـ IP العام، ومجموعة أمان الشبكة (المنافذ 22/80/443)، وبطاقة الشبكة:

```hcl file=./_data/network.tf
```

### الجهاز الافتراضي مع Cloud-Init

يُنشئ `vm.tf` جهاز VM بنظام Ubuntu 24.04. يتم توسيع مسار مفتاح SSH العام عبر `pathexpand()` للتعامل مع `~`. يتلقى قالب cloud-init متغيري `origin_server` و`origin_host`:

```hcl file=./_data/vm.tf
```

### توفير Cloud-Init

يُوفِّر `cloud-init.yaml` الجهاز الافتراضي بضبط النواة، وحدود systemd، وNGINX بتكوين مُحسَّن للأداء، ومنطقة مفاتيح ذاكرة التخزين المؤقت بسعة 128 ميغابايت، ومجموعة اتصالات keepalive للخادم الأصلي، وضغط gzip، وأكثر من 67 رأس لموردي CDN. توفر مكتبة المساعد المشتركة (`/usr/local/lib/cloud-init-helpers.sh`) منطق إعادة المحاولة وتسجيل التقدم إلى `/var/log/cloud-init-progress.log`.

يستخدم cloud-init متغيرات قالب Terraform: `${origin_server}` و`${origin_host}` لتكوين الخادم الأصلي. يتم تخطي متغيرات NGINX مثل `${request_id}` كـ `$${request_id}` في templatefile الخاص بـ Terraform.

```yaml file=./_data/cloud-init.yaml
```

### المخرجات

يعرض `outputs.tf` 17 مخرجاً وفق معيار موارد العرض التوضيحي — 15 مخرجاً قياسياً مشتركاً بين جميع موارد العرض التوضيحي (`deployer`، و`public_ip`، و`private_ip`، و`ssh_command`، و`resource_group_name`، و`vm_name`، و`nsg_name`، و`vnet_name`، و`subnet_id`، و`component`، و`environment`، و`resource_group_id`، و`vm_id`، و`nsg_id`، و`location`) بالإضافة إلى مخرجَين خاصَّين بالمكوّن (`edge_url`، و`health_check_url`):

```hcl file=./_data/outputs.tf
```

### مثال على ملف المتغيرات

انسخ `terraform.tfvars.example` إلى `terraform.tfvars` واملأ قيمك. يستثني `.gitignore` ملف `terraform.tfvars` لمنع إيداع بيانات الاعتماد:

```hcl file=./_data/terraform.tfvars.example
```

## النشر

```bash
# Initialize Terraform
terraform init

# Review the plan
terraform plan

# Apply
terraform apply
```

يعرض Terraform الـ IP العام وأمر SSH وعنوان URL للحافة بعد اكتمال النشر بنجاح.

## حجم الجهاز الافتراضي

يُوصى بأجهزة VM من سلسلة F المُحسَّنة للحوسبة لأعباء عمل NGINX الوكيلة المُكثِّفة لوحدة المعالجة المركزية. الحجم الافتراضي `Standard_F4s_v2` (4 vCPU، 8 جيجابايت) مناسب للاستخدام في المختبر والعرض التوضيحي. تجاوز متغير `vm_size` لسيناريوهات اختبار التحميل أو قياس أداء الإنتاج. يتوسع ضبط نواة cloud-init وتكوين NGINX تلقائياً مع عدد وحدات vCPU (`worker_processes auto`).

## ما بعد النشر

بعد اكتمال `terraform apply`، اسمح بمرور 2-3 دقائق حتى ينتهي cloud-init من تثبيت NGINX وتكوينه. تحقق من نقطة نهاية الفحص الصحي:

```bash
curl -s "http://$(terraform output -raw public_ip)/health" | jq .
```

الاستجابة المتوقعة:

```json
{
  "status": "healthy",
  "component": "cdn-edge",
  "engine": "nginx",
  "vendor_profiles": [
    "akamai",
    "cloudflare",
    "cloudfront",
    "fastly",
    "azure-front-door"
  ]
}
```

## الربط من المكونات الأعلى في السلسلة

يتطلب محاكي CDN وجود خادم مصدر مُنشَر. استخدم مخرجات terraform لخادم المصدر لتعبئة المتغيرات المطلوبة:

```bash
cd ../origin-server/terraform
origin_ip=$(terraform output -raw public_ip)

cd ../../cdn-simulator/terraform
cat > terraform.tfvars <<EOF
subscription_id = "your-subscription-id"
origin_server   = "http://${origin_ip}"
origin_host     = "${origin_ip}:80"
EOF
```

| المتغير المطلوب | المصدر | الصيغة |
|---|---|---|
| `origin_server` | مخرج `public_ip` لخادم المصدر | `http://<ip>` (أدرج المخطط) |
| `origin_host` | مخرج `public_ip` لخادم المصدر | `<ip>:80` (بدون مخطط، أدرج المنفذ) |

انتقل إلى [تكوين NGINX](../04-nginx-config/) لخيارات التخصيص أو [التحقق](../05-verify/) لاختبار ذاكرة التخزين المؤقت.
