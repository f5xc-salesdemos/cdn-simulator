جميع ملفات Terraform موجودة في دليل [`terraform/`](https://github.com/f5xc-salesdemos/cdn-simulator/tree/main/terraform). قم باستنساخ المستودع والنشر مباشرة:

```bash
git clone https://github.com/f5xc-salesdemos/cdn-simulator.git
cd cdn-simulator/terraform
cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars with your Azure subscription ID and origin server
```

## تكوين Terraform

### هيكل ملفات Terraform

يحتوي دليل terraform على 9 ملفات تتبع معيار موارد العرض التوضيحي:

- `versions.tf` — قيود إصدارات Terraform والمزودين (azurerm ~> 4.0, azuread ~> 3.0)
- `providers.tf` — تكوين مزودي Azure RM و Azure AD
- `data.tf` — مصادر بيانات Azure AD للحل التلقائي للناشر
- `locals.tf` — حل الناشر، تسمية الموارد وفق إطار اعتماد السحابة من Azure، العلامات القياسية
- `main.tf` — مجموعة الموارد (المسماة `rg-cdn-simulator-{environment}-{deployer}`)
- `variables.tf` — جميع متغيرات الإدخال (3 مطلوبة، 8 اختيارية)
- `network.tf` — الشبكة الافتراضية (10.100.0.0/16)، الشبكة الفرعية، عنوان IP العام، مجموعة أمان الشبكة (المنافذ 22/80/443)، واجهة الشبكة
- `vm.tf` — جهاز افتراضي بنظام Ubuntu 24.04 مع cloud-init عبر templatefile()
- `outputs.tf` — 17 مخرجاً (15 قياسي + 2 خاص بالمكون)

يعرّف `variables.tf` 11 متغير إدخال منظمة في أقسام عام وحوسبة وخاص بالمكون. يتم حل معرّف `deployer` تلقائياً من حساب Azure AD الخاص بك — تحتاج فقط إلى تعيين `subscription_id` و `origin_server` و `origin_host`:

```hcl file=./_data/variables.tf
```

### البنية التحتية للشبكة

ينشئ `network.tf` الشبكة الافتراضية والشبكة الفرعية وعنوان IP العام ومجموعة أمان الشبكة (المنافذ 22/80/443) وواجهة الشبكة:

```hcl file=./_data/network.tf
```

### الجهاز الافتراضي مع Cloud-Init

ينشئ `vm.tf` جهازاً افتراضياً بنظام Ubuntu 24.04. يتم توسيع مسار المفتاح العام لـ SSH عبر `pathexpand()` للتعامل مع `~`. يتلقى قالب cloud-init متغيرات `origin_server` و `origin_host`:

```hcl file=./_data/vm.tf
```

### التجهيز عبر Cloud-Init

يقوم `cloud-init.yaml` بتجهيز الجهاز الافتراضي مع ضبط النواة وحدود systemd وتكوين NGINX المحسّن للأداء ومنطقة مفاتيح التخزين المؤقت بحجم 128 ميجابايت ومجمع اتصالات upstream المستمرة وضغط gzip وأكثر من 67 ترويسة من مزودي CDN. توفر مكتبة مساعدة مشتركة (`/usr/local/lib/cloud-init-helpers.sh`) منطق إعادة المحاولة وتسجيل التقدم في `/var/log/cloud-init-progress.log`.

يستخدم cloud-init متغيرات قالب Terraform: `${origin_server}` و `${origin_host}` لتكوين upstream. يتم تهريب متغيرات NGINX مثل `${request_id}` كـ `$${request_id}` في templatefile الخاص بـ Terraform.

```yaml file=./_data/cloud-init.yaml
```

### المخرجات

يعرض `outputs.tf` 17 مخرجاً وفق معيار موارد العرض التوضيحي — 15 مخرجاً قياسياً مشتركاً بين جميع موارد العرض التوضيحي (`deployer`، `public_ip`، `private_ip`، `ssh_command`، `resource_group_name`، `vm_name`، `nsg_name`، `vnet_name`، `subnet_id`، `component`، `environment`، `resource_group_id`، `vm_id`، `nsg_id`، `location`) بالإضافة إلى مخرجين خاصين بالمكون (`edge_url`، `health_check_url`):

```hcl file=./_data/outputs.tf
```

### ملف المتغيرات النموذجي

انسخ `terraform.tfvars.example` إلى `terraform.tfvars` واملأ القيم الخاصة بك. يستثني `.gitignore` ملف `terraform.tfvars` لمنع رفع بيانات الاعتماد:

```hcl file=./_data/terraform.tfvars.example
```

## النشر

```bash
# Initialize Terraform
terraform init

# Review the plan
terraform plan

# Apply
terraform apply
```

يعرض Terraform عنوان IP العام وأمر SSH وعنوان URL الطرفي بعد النشر الناجح.

## حجم الجهاز الافتراضي

يُوصى باستخدام الأجهزة الافتراضية من فئة F المحسّنة للحوسبة لهذا الحمل العملي لوكيل NGINX المعتمد على المعالج. الحجم الافتراضي `Standard_F4s_v2` (4 معالجات افتراضية، 8 جيبي بايت) مناسب للاستخدام المختبري والعروض التوضيحية. قم بتجاوز متغير `vm_size` لسيناريوهات اختبار الحمل أو قياس أداء الإنتاج. يتكيف ضبط النواة عبر cloud-init وتكوين NGINX تلقائياً مع عدد المعالجات الافتراضية (`worker_processes auto`).

## ما بعد النشر

بعد اكتمال `terraform apply`، انتظر 2-3 دقائق حتى ينتهي cloud-init من تثبيت وتكوين NGINX. تحقق من نقطة نهاية الصحة:

```bash
curl -s "http://$(terraform output -raw public_ip)/health" | jq .
```

الاستجابة المتوقعة:

```json
{
  "status": "healthy",
  "component": "cdn-edge",
  "engine": "nginx",
  "vendor_profiles": [
    "akamai",
    "cloudflare",
    "cloudfront",
    "fastly",
    "azure-front-door"
  ]
}
```

## الربط من المكونات العلوية

يتطلب محاكي CDN خادم أصل منشوراً. استخدم مخرجات terraform لخادم الأصل لملء المتغيرات المطلوبة:

```bash
cd ../origin-server/terraform
origin_ip=$(terraform output -raw public_ip)

cd ../../cdn-simulator/terraform
cat > terraform.tfvars <<EOF
subscription_id = "your-subscription-id"
origin_server   = "http://${origin_ip}"
origin_host     = "${origin_ip}:80"
EOF
```

| المتغير المطلوب | المصدر | التنسيق |
|---|---|---|
| `origin_server` | مخرج `public_ip` لخادم الأصل | `http://<ip>` (يشمل المخطط) |
| `origin_host` | مخرج `public_ip` لخادم الأصل | `<ip>:80` (بدون مخطط، مع المنفذ) |

انتقل إلى [تكوين NGINX](../04-nginx-config/) لخيارات التخصيص أو [التحقق](../05-verify/) لاختبار التخزين المؤقت.
