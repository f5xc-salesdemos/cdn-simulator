تغطي هذه الصفحة مرحلتين من التكامل:

1. **التكامل المباشر** — حافة CDN تُوجّه الطلبات إلى خادم الأصل مباشرة (اختبار خط الأساس)
2. **إدراج F5 XC** — يتم إدراج موازن حمل HTTP الخاص بـ F5 XC بين CDN وخادم الأصل (عرض الأمان)

ابدأ بالتكامل المباشر لإنشاء خط الأساس، ثم أدرج F5 XC عندما تكون جاهزًا.

## مرجع خادم الأصل

يوفر مكوّن المختبر [origin-server](https://f5xc-salesdemos.github.io/origin-server/) تطبيقات ويب معرّضة للثغرات لاختبار الأمان.

| الخاصية | القيمة |
|---|---|
| التوثيق | [f5xc-salesdemos.github.io/origin-server](https://f5xc-salesdemos.github.io/origin-server/) |
| المستودع | [github.com/f5xc-salesdemos/origin-server](https://github.com/f5xc-salesdemos/origin-server) |
| المنفذ الافتراضي | 80 |
| فحص الحالة | `GET /health` |

### التطبيقات المتاحة

احصل على كتالوج التطبيقات الحالي من بيان نقاط النهاية المنشور لخادم الأصل:

```bash
MANIFEST_URL=$(curl -sf https://api.github.com/repos/f5xc-salesdemos/origin-server/releases/latest \
  | python3 -c "import sys,json; assets=json.load(sys.stdin).get('assets',[]); print(next((a['browser_download_url'] for a in assets if a['name']=='manifest.json'),''))")
curl -sf "$MANIFEST_URL" | python3 -m json.tool
```

إذا لم يوجد إصدار بعد، استخدم مصدر المستودع مباشرة:

```bash
curl -sf https://raw.githubusercontent.com/f5xc-salesdemos/origin-server/main/manifest.json | python3 -m json.tool
```

يسرد البيان جميع مسارات التطبيقات وفحوصات الحالة وصور الحاويات وربط ميزات العرض التوضيحي.

### مسارات التطبيقات

| المسار | التطبيق | ميزات العرض التوضيحي |
|---|---|---|
| `/` | صفحة الهبوط | — |
| `/health` | فحص الحالة | — |
| `/juice-shop/` | OWASP Juice Shop | WAF، الحماية من الروبوتات، أمان API |
| `/dvwa/` | DVWA | WAF، الحماية من الروبوتات |
| `/vampi/` | VAmPI | أمان API |
| `/httpbin/` | httpbin | التشخيصات |
| `/whoami/` | whoami | التحقق من الترويسات |
| `/csd-demo/` | CSD Demo | الدفاع من جانب العميل |

## المرحلة 1: التكامل المباشر (خط الأساس)

```
┌──────────┐     ┌──────────────────────┐     ┌─────────────────────┐
│  العميل   │────▶│  حافة CDN (NGINX)    │────▶│  خادم الأصل         │
│           │     │  20.65.90.112        │     │  20.12.78.159       │
└──────────┘     │  67+ ترويسة CDN      │     │  Juice Shop, DVWA,  │
                 │  ذاكرة تخزين مؤقت    │     │  VAmPI, httpbin,    │
                 └──────────────────────┘     │  whoami, CSD Demo   │
                                              └─────────────────────┘
```

### تهيئة خادم الأصل

عيّن عنوان IP لخادم الأصل في تهيئة NGINX الخاصة بحافة CDN:

```bash
ssh azureuser@<CDN_EDGE_IP>
sudo sed -i 's|proxy_pass .*;|proxy_pass http://<ORIGIN_IP>;|' /etc/nginx/conf.d/cdn-edge.conf
sudo rm -rf /var/cache/nginx/cdn/*
sudo nginx -t && sudo systemctl reload nginx
```

أو عيّنه في وقت نشر Terraform عبر `terraform.tfvars`:

```hcl
origin_server = "http://<ORIGIN_IP>"
```

### التحقق من جميع التطبيقات

اختبر كل تطبيق أصل عبر CDN:

```bash
CDN=<CDN_EDGE_IP>

# Health check (CDN local)
curl -sf "http://$CDN/health" | python3 -m json.tool

# Landing page
curl -sf -o /dev/null -w "/ : HTTP %{http_code}\n" "http://$CDN/"

# Juice Shop
curl -sf -o /dev/null -w "/juice-shop/ : HTTP %{http_code}\n" "http://$CDN/juice-shop/"

# DVWA (follows redirect to login)
curl -sf -o /dev/null -w "/dvwa/ : HTTP %{http_code}\n" -L "http://$CDN/dvwa/"

# VAmPI API
curl -sf "http://$CDN/vampi/users/v1" | python3 -m json.tool | head -5

# httpbin headers (shows CDN headers in JSON)
curl -sf "http://$CDN/httpbin/headers" | python3 -m json.tool | head -10

# whoami (shows ALL headers the origin receives)
curl -sf "http://$CDN/whoami/"

# CSD Demo
curl -sf -o /dev/null -w "/csd-demo/ : HTTP %{http_code}\n" "http://$CDN/csd-demo/"
```

يجب أن تُرجع جميع المسارات HTTP 200 (DVWA يُرجع 302 ثم 200 عند المتابعة).

### التحقق من ترويسات CDN عند خادم الأصل

تعرض نقطة النهاية `/whoami/` كل ترويسة يتلقاها خادم الأصل. عند الوصول إليها عبر CDN، تعرض جميع الترويسات الـ 67+ الخاصة بالمزودين:

```bash
curl -sf "http://$CDN/whoami/"
```

تحقق من وجود هذه الترويسات الرئيسية:

| المزوّد | الترويسة | القيمة المتوقعة |
|---|---|---|
| قياسي | `X-Forwarded-For` | `<your_ip>, <cdn_edge_ip>` |
| Akamai | `True-Client-Ip` | `<your_ip>` |
| Cloudflare | `Cf-Connecting-Ip` | `<your_ip>` |
| CloudFront | `Cloudfront-Viewer-Country` | `US` |
| Fastly | `Fastly-Client-Ip` | `<your_ip>` |
| Azure FD | `X-Azure-Clientip` | `<your_ip>` |

### مطابقة السجلات

قارن سجلات الوصول على كلا الخادمين للتحقق من تدفق حركة المرور:

```bash
# CDN edge log — shows your client IP as source
ssh azureuser@<CDN_EDGE_IP> "sudo tail -5 /var/log/nginx/access.log"

# Origin log — shows CDN edge IP as source
ssh azureuser@<ORIGIN_IP> "sudo tail -5 /var/log/nginx/access.log"
```

يجب أن يُظهر سجل خادم الأصل عنوان IP الخاص بحافة CDN كعميل متصل، بينما يُحمل عنوان IP الحقيقي للعميل في `X-Forwarded-For` والترويسات الخاصة بالمزود.

### سلوك التخزين المؤقت

```bash
# First request — MISS (fetched from origin)
curl -s -I "http://$CDN/whoami/" | grep X-Cache-Status

# Second request — HIT (served from CDN cache)
curl -s -I "http://$CDN/whoami/" | grep X-Cache-Status
```

## المرحلة 2: إدراج F5 XC (عرض الأمان)

بعد اختبار خط الأساس، أدرج موازن حمل HTTP الخاص بـ F5 XC بين CDN وخادم الأصل:

```
┌──────────┐     ┌────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│  العميل   │────▶│  حافة CDN      │────▶│  F5 XC HTTP LB   │────▶│  خادم الأصل     │
│           │     │  (NGINX)       │     │  WAF + Bot + API  │     │                 │
└──────────┘     └────────────────┘     └──────────────────┘     └─────────────────┘
```

1. **أنشئ موازن حمل HTTP الخاص بـ F5 XC** مع خادم الأصل في مجموعة الأصول الخاصة به
2. **حدّث حافة CDN** لتوجيه الطلبات إلى VIP الخاص بـ F5 XC بدلاً من خادم الأصل مباشرة:

```bash
ssh azureuser@<CDN_EDGE_IP>
sudo sed -i 's|proxy_pass .*;|proxy_pass https://<F5XC_LB_VIP>;|' /etc/nginx/conf.d/cdn-edge.conf
sudo rm -rf /var/cache/nginx/cdn/*
sudo nginx -t && sudo systemctl reload nginx
```

3. **تحقق من تطبيق WAF** عبر السلسلة الكاملة:

```bash
# SQL injection through CDN → F5 XC WAF should block
curl -I "http://$CDN/dvwa/vulnerabilities/sqli/?id=%27+OR+1%3D1--"

# Normal request should pass
curl -sf -o /dev/null -w "%{http_code}" "http://$CDN/juice-shop/"
```

4. **هيّئ ترويسة عنوان IP الموثوق للعميل** في F5 XC لقراءة عنوان IP الحقيقي للعميل من ترويسات CDN (مثل `True-Client-IP` لمحاكاة Akamai، أو `CF-Connecting-IP` لمحاكاة Cloudflare)
