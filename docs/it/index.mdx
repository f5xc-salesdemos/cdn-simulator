## Cosa Offre

- **[Singolo Nodo Edge](01-overview/)** — Una VM Ubuntu 24.04 che esegue NGINX come reverse proxy con cache, il minimo CDN edge funzionante
- **[Distribuzione Terraform](03-deploy/)** — Infrastruttura Azure completa come codice: VNet, PIP, NSG e VM con provisioning cloud-init
- **[67+ Header di Vendor CDN](04-nginx-config/)** — Simula gli header di richiesta di Akamai, Cloudflare, CloudFront, Fastly e Azure Front Door, inclusi True-Client-IP, CF-Connecting-IP, geolocalizzazione, rilevamento dispositivo, fingerprint JA3/JA4 e punteggi bot
- **[Integrazione Origin](06-integrate/)** — Configura l'edge per inoltrare i cache miss a un bilanciatore di carico HTTP F5 XC come origin
- **[Verifica della Cache](05-verify/)** — Valida il comportamento HIT/MISS con gli header X-Cache-Status e l'ispezione della cache NGINX

## Riferimento Rapido

Dopo la distribuzione, il nodo edge CDN espone:

| Endpoint | Metodo | Scopo |
|---|---|---|
| `http://<PUBLIC_IP>/health` | GET | Health check — restituisce JSON con stato dei componenti e profili vendor |
| `http://<PUBLIC_IP>/` | GET | Proxy CDN — mette in cache e inoltra le richieste al server origin |
| `http://<PUBLIC_IP>/<any-path>` | GET | Proxy CDN — tutti i percorsi vengono inoltrati tramite proxy con cache e header vendor |

Header di risposta aggiunti a ogni risposta inoltrata tramite proxy:

| Header | Esempio | Scopo |
|---|---|---|
| `X-Cache-Status` | `HIT` o `MISS` | Indica se la risposta è stata servita dalla cache |
| `X-CDN-Edge` | `cdn-simulator` | Identifica questo nodo edge |
| `X-CDN-POP` | `SJC` | Codice simulato del Point of Presence |
| `X-Served-By` | `cache-sjc3120-SJC` | Identificatore simulato del nodo cache |
| `X-Request-ID` | UUID | Identificatore univoco per richiesta |
