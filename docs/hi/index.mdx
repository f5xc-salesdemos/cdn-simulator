## यह क्या प्रदान करता है

- **[सिंगल एज नोड](01-overview/)** — NGINX को कैशिंग रिवर्स प्रॉक्सी के रूप में चलाने वाला एक Ubuntu 24.04 VM, न्यूनतम व्यवहार्य CDN एज
- **[Terraform डिप्लॉयमेंट](03-deploy/)** — संपूर्ण Azure इन्फ्रास्ट्रक्चर एज़ कोड: VNet, PIP, NSG, और cloud-init प्रोविज़निंग के साथ VM
- **[67+ CDN वेंडर हेडर](04-nginx-config/)** — Akamai, Cloudflare, CloudFront, Fastly, और Azure Front Door रिक्वेस्ट हेडर का अनुकरण करता है जिसमें True-Client-IP, CF-Connecting-IP, जियोलोकेशन, डिवाइस डिटेक्शन, JA3/JA4 फिंगरप्रिंट्स, और बॉट स्कोर शामिल हैं
- **[ओरिजिन इंटीग्रेशन](06-integrate/)** — कैश मिस को ओरिजिन के रूप में F5 XC HTTP लोड बैलेंसर पर फॉरवर्ड करने के लिए एज को कॉन्फ़िगर करें
- **[कैश वेरिफिकेशन](05-verify/)** — X-Cache-Status हेडर और NGINX कैश इंस्पेक्शन के साथ HIT/MISS व्यवहार को वैलिडेट करें

## त्वरित संदर्भ

डिप्लॉयमेंट के बाद, CDN एज नोड निम्नलिखित एक्सपोज़ करता है:

| एंडपॉइंट | मेथड | उद्देश्य |
|---|---|---|
| `http://<PUBLIC_IP>/health` | GET | हेल्थ चेक — कंपोनेंट स्टेटस और वेंडर प्रोफाइल के साथ JSON लौटाता है |
| `http://<PUBLIC_IP>/` | GET | CDN प्रॉक्सी — रिक्वेस्ट को कैश करता है और ओरिजिन सर्वर पर फॉरवर्ड करता है |
| `http://<PUBLIC_IP>/<any-path>` | GET | CDN प्रॉक्सी — सभी पथ कैशिंग और वेंडर हेडर के साथ प्रॉक्सी किए जाते हैं |

हर प्रॉक्सी किए गए रिस्पॉन्स में जोड़े गए रिस्पॉन्स हेडर:

| हेडर | उदाहरण | उद्देश्य |
|---|---|---|
| `X-Cache-Status` | `HIT` या `MISS` | क्या रिस्पॉन्स कैश से सर्व किया गया |
| `X-CDN-Edge` | `cdn-simulator` | इस एज नोड की पहचान करता है |
| `X-CDN-POP` | `SJC` | सिम्युलेटेड पॉइंट ऑफ प्रेज़ेंस कोड |
| `X-Served-By` | `cache-sjc3120-SJC` | सिम्युलेटेड कैश नोड आइडेंटिफायर |
| `X-Request-ID` | UUID | प्रति-रिक्वेस्ट यूनिक आइडेंटिफायर |
