## यह क्या प्रदान करता है

- **[सिंगल एज नोड](01-overview/)** — एक Ubuntu 24.04 VM जो NGINX को कैशिंग रिवर्स प्रॉक्सी के रूप में चलाता है, न्यूनतम व्यवहार्य CDN एज
- **[Terraform डिप्लॉयमेंट](03-deploy/)** — पूर्ण Azure इन्फ्रास्ट्रक्चर ऐज़ कोड: VNet, PIP, NSG, और cloud-init प्रोविज़निंग के साथ VM
- **[67+ CDN वेंडर हेडर्स](04-nginx-config/)** — Akamai, Cloudflare, CloudFront, Fastly, और Azure Front Door रिक्वेस्ट हेडर्स का अनुकरण करता है जिसमें True-Client-IP, CF-Connecting-IP, जियोलोकेशन, डिवाइस डिटेक्शन, JA3/JA4 फिंगरप्रिंट्स, और बॉट स्कोर शामिल हैं
- **[ऑरिजिन इंटीग्रेशन](06-integrate/)** — एज को कॉन्फ़िगर करें ताकि कैश मिस को ऑरिजिन के रूप में F5 XC HTTP लोड बैलेंसर पर फ़ॉरवर्ड किया जा सके
- **[कैश वेरिफ़िकेशन](05-verify/)** — X-Cache-Status हेडर्स और NGINX कैश इंस्पेक्शन के साथ HIT/MISS व्यवहार को सत्यापित करें

## त्वरित संदर्भ

डिप्लॉयमेंट के बाद, CDN एज नोड निम्नलिखित एक्सपोज़ करता है:

| एंडपॉइंट | मेथड | उद्देश्य |
|---|---|---|
| `http://<PUBLIC_IP>/health` | GET | हेल्थ चेक — कंपोनेंट स्टेटस और वेंडर प्रोफ़ाइल के साथ JSON लौटाता है |
| `http://<PUBLIC_IP>/` | GET | CDN प्रॉक्सी — रिक्वेस्ट को कैश करता है और ऑरिजिन सर्वर पर फ़ॉरवर्ड करता है |
| `http://<PUBLIC_IP>/<any-path>` | GET | CDN प्रॉक्सी — सभी पाथ कैशिंग और वेंडर हेडर्स के साथ प्रॉक्सी किए जाते हैं |

प्रत्येक प्रॉक्सी किए गए रिस्पॉन्स में जोड़े जाने वाले रिस्पॉन्स हेडर्स:

| हेडर | उदाहरण | उद्देश्य |
|---|---|---|
| `X-Cache-Status` | `HIT` या `MISS` | रिस्पॉन्स कैश से सर्व किया गया था या नहीं |
| `X-CDN-Edge` | `cdn-simulator` | इस एज नोड की पहचान करता है |
| `X-CDN-POP` | `SJC` | अनुकरित पॉइंट ऑफ़ प्रेज़ेंस कोड |
| `X-Served-By` | `cache-sjc3120-SJC` | अनुकरित कैश नोड आइडेंटिफ़ायर |
| `X-Request-ID` | UUID | प्रति-रिक्वेस्ट यूनिक आइडेंटिफ़ायर |
