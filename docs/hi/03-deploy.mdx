सभी Terraform फाइलें [`terraform/`](https://github.com/f5xc-salesdemos/cdn-simulator/tree/main/terraform) डायरेक्टरी में हैं। रिपॉजिटरी क्लोन करें और सीधे डिप्लॉय करें:

```bash
git clone https://github.com/f5xc-salesdemos/cdn-simulator.git
cd cdn-simulator/terraform
cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars with your Azure subscription ID and origin server
```

## Terraform कॉन्फ़िगरेशन

### Terraform फाइल संरचना

terraform डायरेक्टरी में Demo Resource Standard का पालन करते हुए 9 फाइलें हैं:

- `versions.tf` — Terraform और प्रोवाइडर वर्शन कंस्ट्रेंट्स (azurerm ~> 4.0, azuread ~> 3.0)
- `providers.tf` — Azure RM और Azure AD प्रोवाइडर कॉन्फ़िगरेशन
- `data.tf` — डिप्लॉयर ऑटो-रिज़ॉल्यूशन के लिए Azure AD डेटा सोर्स
- `locals.tf` — डिप्लॉयर रिज़ॉल्यूशन, Azure Cloud Adoption Framework रिसोर्स नेमिंग, स्टैंडर्ड टैग
- `main.tf` — रिसोर्स ग्रुप (`rg-cdn-simulator-{environment}-{deployer}` नाम से)
- `variables.tf` — सभी इनपुट वेरिएबल्स (3 आवश्यक, 8 वैकल्पिक)
- `network.tf` — VNet (10.100.0.0/16), सबनेट, पब्लिक IP, NSG (पोर्ट 22/80/443), NIC
- `vm.tf` — templatefile() के माध्यम से cloud-init के साथ Ubuntu 24.04 VM
- `outputs.tf` — 17 आउटपुट (15 स्टैंडर्ड + 2 कम्पोनेंट-विशिष्ट)

`variables.tf` में General, Compute, और Component-Specific सेक्शन में व्यवस्थित 11 इनपुट वेरिएबल्स परिभाषित हैं। `deployer` आइडेंटिफायर आपके Azure AD अकाउंट से ऑटो-रिज़ॉल्व होता है — आपको केवल `subscription_id`, `origin_server`, और `origin_host` सेट करने की आवश्यकता है:

```hcl file=./_data/variables.tf
```

### नेटवर्क इंफ्रास्ट्रक्चर

`network.tf` VNet, सबनेट, पब्लिक IP, NSG (पोर्ट 22/80/443), और NIC बनाता है:

```hcl file=./_data/network.tf
```

### Cloud-Init के साथ वर्चुअल मशीन

`vm.tf` Ubuntu 24.04 VM बनाता है। SSH पब्लिक की पाथ को `~` हैंडल करने के लिए `pathexpand()` के माध्यम से एक्सपैंड किया जाता है। Cloud-init टेम्पलेट `origin_server` और `origin_host` वेरिएबल्स प्राप्त करता है:

```hcl file=./_data/vm.tf
```

### Cloud-Init प्रोविजनिंग

`cloud-init.yaml` VM को कर्नल ट्यूनिंग, systemd लिमिट्स, परफॉर्मेंस-ऑप्टिमाइज़्ड कॉन्फ़िगरेशन के साथ NGINX, 128 MB कैश कीज़ ज़ोन, अपस्ट्रीम कीपअलाइव पूल, gzip कम्प्रेशन, और 67+ CDN वेंडर हेडर्स के साथ प्रोविजन करता है। एक साझा हेल्पर लाइब्रेरी (`/usr/local/lib/cloud-init-helpers.sh`) `/var/log/cloud-init-progress.log` में रीट्राई लॉजिक और प्रोग्रेस लॉगिंग प्रदान करती है।

Cloud-init अपस्ट्रीम कॉन्फ़िगरेशन के लिए Terraform टेम्पलेट वेरिएबल्स का उपयोग करता है: `${origin_server}` और `${origin_host}`। NGINX वेरिएबल्स जैसे `${request_id}` को Terraform templatefile में `$${request_id}` के रूप में एस्केप किया जाता है।

```yaml file=./_data/cloud-init.yaml
```

### आउटपुट

`outputs.tf` Demo Resource Standard का पालन करते हुए 17 आउटपुट एक्सपोज़ करता है — 15 स्टैंडर्ड आउटपुट जो सभी डेमो रिसोर्सेज़ द्वारा साझा किए जाते हैं (`deployer`, `public_ip`, `private_ip`, `ssh_command`, `resource_group_name`, `vm_name`, `nsg_name`, `vnet_name`, `subnet_id`, `component`, `environment`, `resource_group_id`, `vm_id`, `nsg_id`, `location`) और 2 कम्पोनेंट-विशिष्ट आउटपुट (`edge_url`, `health_check_url`):

```hcl file=./_data/outputs.tf
```

### उदाहरण वेरिएबल्स फाइल

`terraform.tfvars.example` को `terraform.tfvars` में कॉपी करें और अपने मान भरें। `.gitignore` क्रेडेंशियल्स को कमिट होने से रोकने के लिए `terraform.tfvars` को एक्सक्लूड करता है:

```hcl file=./_data/terraform.tfvars.example
```

## डिप्लॉय

```bash
# Initialize Terraform
terraform init

# Review the plan
terraform plan

# Apply
terraform apply
```

सफल डिप्लॉयमेंट के बाद Terraform पब्लिक IP, SSH कमांड, और एज URL आउटपुट करता है।

## VM साइज़िंग

इस CPU-बाउंड NGINX प्रॉक्सी वर्कलोड के लिए F-series कम्प्यूट-ऑप्टिमाइज़्ड VM की अनुशंसा की जाती है। डिफ़ॉल्ट `Standard_F4s_v2` (4 vCPU, 8 GiB) लैब और डेमो उपयोग के लिए उपयुक्त है। लोड टेस्टिंग या प्रोडक्शन बेंचमार्किंग परिदृश्यों के लिए `vm_size` वेरिएबल को ओवरराइड करें। Cloud-init कर्नल ट्यूनिंग और NGINX कॉन्फ़िग vCPU काउंट के साथ स्वचालित रूप से स्केल होते हैं (`worker_processes auto`)।

## डिप्लॉय के बाद

`terraform apply` पूरा होने के बाद, cloud-init द्वारा NGINX की इंस्टॉलेशन और कॉन्फ़िगरेशन पूरी होने के लिए 2-3 मिनट का समय दें। हेल्थ एंडपॉइंट को वेरिफाई करें:

```bash
curl -s "http://$(terraform output -raw public_ip)/health" | jq .
```

अपेक्षित रिस्पॉन्स:

```json
{
  "status": "healthy",
  "component": "cdn-edge",
  "engine": "nginx",
  "vendor_profiles": [
    "akamai",
    "cloudflare",
    "cloudfront",
    "fastly",
    "azure-front-door"
  ]
}
```

## अपस्ट्रीम कम्पोनेंट्स से वायरिंग

CDN सिमुलेटर को एक डिप्लॉय किए गए ऑरिजिन सर्वर की आवश्यकता है। आवश्यक वेरिएबल्स को भरने के लिए ऑरिजिन सर्वर के terraform आउटपुट का उपयोग करें:

```bash
cd ../origin-server/terraform
origin_ip=$(terraform output -raw public_ip)

cd ../../cdn-simulator/terraform
cat > terraform.tfvars <<EOF
subscription_id = "your-subscription-id"
origin_server   = "http://${origin_ip}"
origin_host     = "${origin_ip}:80"
EOF
```

| आवश्यक वेरिएबल | स्रोत | फॉर्मेट |
|---|---|---|
| `origin_server` | ऑरिजिन सर्वर `public_ip` आउटपुट | `http://<ip>` (स्कीम शामिल करें) |
| `origin_host` | ऑरिजिन सर्वर `public_ip` आउटपुट | `<ip>:80` (स्कीम नहीं, पोर्ट शामिल करें) |

कस्टमाइज़ेशन विकल्पों के लिए [NGINX कॉन्फ़िगरेशन](../04-nginx-config/) पर जाएं या कैश टेस्टिंग के लिए [वेरिफाई](../05-verify/) पर जाएं।
